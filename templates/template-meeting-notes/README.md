# Meeting Notes (with Zoom)

An agent + workflow that turns a Zoom cloud recording into structured meeting notes — attendees, decisions, action items, topics — and (optionally) ships those notes to Notion or Linear. Built on Mastra with the Mastra Gateway and Turso (libSQL).

## Why we built this

Most meetings end with a clip of audio nobody re-listens to. This template wires the boring plumbing for you:

- Zoom Server-to-Server OAuth (one-time setup, no per-user OAuth dance).
- A workflow that pulls the Zoom-generated VTT transcript for a meeting, summarizes it with `gpt-5-mini` via the Mastra Gateway, and returns structured JSON.
- Optional one-line exports to Notion and Linear, gated on env vars so they're easy to leave disabled.
- A `POST /meetings/upload` API route for transcripts that don't come from Zoom (Google Meet, Riverside, Loom, plain text).

## Demo

This demo runs in Mastra Studio. Open the `meetingNotesAgent` and ask it to summarize a transcript, or run the `ingest-meeting` workflow with a Zoom meeting id.

## Prerequisites

- A [Mastra Gateway API key](https://mastra.ai/docs/models/gateways/mastra).
- A [Turso](https://turso.tech) database URL + auth token.
- A Zoom **Server-to-Server OAuth** app. See setup below.
- (Optional) A [Linear](https://linear.app) API key to create issues from action items.
- (Optional) A [Notion](https://notion.so) integration token + a parent page id to write notes to.

## Quickstart 🚀

1. **Clone the template**
   - Run `npx create-mastra@latest --template meeting-notes`.
2. **Set up a Zoom Server-to-Server OAuth app**
   - At https://marketplace.zoom.us → "Develop" → "Build App" → choose "Server-to-Server OAuth".
   - Add these scopes:
     - `cloud_recording:read:list_user_recordings:admin`
     - `cloud_recording:read:recording:admin`
     - `user:read:user:admin`
   - Activate the app and copy the **Account ID**, **Client ID**, and **Client Secret** into `.env`.
3. **Add your API keys**
   - Copy `.env.example` to `.env` and fill it in.
4. **Start the dev server**
   - Run `npm run dev` and open [localhost:4111](http://localhost:4111).

Then try:

- In Studio, run the `ingest-meeting` workflow with a Zoom `meetingId`.
- Or chat with the `meetingNotesAgent`: "List my recent Zoom recordings", then "Summarize the one from Tuesday".

## Zoom transcripts

This template uses Zoom's built-in transcript (generated by Zoom AI Companion). Make sure the cloud-recording setting "Audio transcript" is enabled for the user whose meetings you're ingesting. If a meeting has no Zoom transcript, `fetchZoomTranscript` will throw — wire in your own STT service (e.g. Whisper, Deepgram, AssemblyAI) as a fallback.

## Manual upload endpoint

```http
POST /meetings/upload
Content-Type: application/json

{
  "title": "Q3 planning sync",
  "transcript": "Alice: Let's start with..."
}
```

Returns the structured notes JSON. Useful for Google Meet / Riverside / Loom / plain text sources.

## Making it yours

- **Add Whisper fallback.** Drop a `transcribeAudio` tool that downloads the recording's audio file and runs it through Whisper, so meetings without Zoom AI still work.
- **Tighten the output schema.** Edit `meetingNotesOutputSchema` to capture the fields your team actually uses (e.g. risks, blockers, follow-up meetings).
- **Auto-create Linear issues.** Add a workflow step that loops over `actionItems` and calls `createLinearIssue` for each one.
- **Trigger on Zoom webhooks.** Add an API route at `/zoom/webhook` that listens for the `recording.completed` event and kicks off the `ingest-meeting` workflow automatically.
