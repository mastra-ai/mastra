<div id="mcp-overview">
  # MCP の概要
</div>

Mastra は、AI エージェントを外部のツールやリソースに接続するためのオープン標準である [Model Context Protocol (MCP)](https://modelcontextprotocol.io/introduction) をサポートしています。汎用プラグインシステムとして機能し、言語やホスティング環境に関係なくエージェントがツールを呼び出せるようにします。

Mastra は MCP サーバーの作成にも使用でき、MCP インターフェースを通じてエージェント、ツール、その他の構造化リソースを公開します。これらは、プロトコルをサポートするあらゆるシステムやエージェントからアクセスできます。

Mastra は現在、次の 2 つの MCP クラスをサポートしています:

1. **`MCPClient`**: 1 つまたは複数の MCP サーバーに接続し、それらのツール、リソース、プロンプトにアクセスし、エリシテーション（情報引き出し）リクエストを処理します。
2. **`MCPServer`**: Mastra のツール、エージェント、ワークフロー、プロンプト、リソースを MCP 互換クライアントに公開します。

<div id="getting-started">
  ## はじめに
</div>

MCP を使い始めるには、必要な依存関係をインストールします。

```bash
npm install @mastra/mcp@latest
```


<div id="configuring-mcpclient">
  ## `MCPClient` の設定
</div>

`MCPClient` は Mastra のプリミティブを外部の MCP サーバーに接続します。サーバーは、`npx` を使って起動するローカルパッケージか、リモートの HTTP(S) エンドポイントのいずれかです。各サーバーは、ホスティング形態に応じて `command` または `url` のいずれかを指定して設定する必要があります。

```typescript title="src/mastra/mcp/test-mcp-client.ts" showLineNumbers copy
import { MCPClient } from "@mastra/mcp";

export const testMcpClient = new MCPClient({
  id: "test-mcp-client",
  servers: {
    wikipedia: {
      command: "npx",
      args: ["-y", "wikipedia-mcp"],
    },
    weather: {
      url: new URL(
        `https://server.smithery.ai/@smithery-ai/national-weather-service/mcp?api_key=${process.env.SMITHERY_API_KEY}`,
      ),
    },
  },
});
```

> すべての設定オプションの一覧は [MCPClient](/reference/tools/mcp-client) を参照してください。


<div id="using-mcpclient-with-an-agent">
  ## エージェントでの `MCPClient` の使用
</div>

エージェントで MCP サーバーのツールを使うには、`MCPClient` をインポートし、`tools` パラメータで `.getTools()` を呼び出します。これにより、定義済みの MCP サーバーからツールが読み込まれ、エージェントで利用可能になります。

```typescript {4,16} title="src/mastra/agents/test-agent.ts" showLineNumbers copy
import { openai } from "@ai-sdk/openai";
import { Agent } from "@mastra/core/agent";

import { testMcpClient } from "../mcp/test-mcp-client";

export const testAgent = new Agent({
  name: "テストエージェント",
  description: "あなたは役に立つAIアシスタントです",
  instructions: `
      あなたは以下のMCPサーバーにアクセス可能な役に立つアシスタントです。
      - Wikipedia MCPサーバー
      - 米国国立気象局

      MCPサーバーを使用して取得した情報を基に質問に答えてください。`,
  model: openai("gpt-4o-mini"),
  tools: await testMcpClient.getTools(),
});
```

> すべての設定オプションについては、[Agent Class](/reference/agents/agent) を参照してください。


<div id="configuring-mcpserver">
  ## `MCPServer` の設定
</div>

Mastra アプリのエージェント、ツール、ワークフローを HTTP(S) 経由で外部システムに公開するには、`MCPServer` クラスを使用します。これにより、当該プロトコルに対応するあらゆるシステムやエージェントから利用可能になります。

```typescript title="src/mastra/mcp/test-mcp-server.ts" showLineNumbers copy
import { MCPServer } from "@mastra/mcp";

import { testAgent } from "../agents/test-agent";
import { testWorkflow } from "../workflows/test-workflow";
import { testTool } from "../tools/test-tool";

export const testMcpServer = new MCPServer({
  id: "test-mcp-server",
  name: "テストサーバー",
  version: "1.0.0",
  agents: { testAgent },
  tools: { testTool },
  workflows: { testWorkflow },
});
```

> すべての設定オプションの一覧については、[MCPServer](/reference/tools/mcp-server) を参照してください。


<div id="registering-an-mcpserver">
  ## `MCPServer` の登録
</div>

プロトコル対応の他のシステムやエージェントで MCP サーバーを利用できるようにするには、メインの `Mastra` インスタンスで `mcpServers` を使用して登録します。

```typescript title="src/mastra/index.ts" showLineNumbers copy
import { Mastra } from "@mastra/core/mastra";

import { testMcpServer } from "./mcp/test-mcp-server";

export const mastra = new Mastra({
  // ...
  mcpServers: { testMcpServer },
});
```


<div id="static-and-dynamic-tools">
  ## 静的ツールと動的ツール
</div>

`MCPClient` は、接続先サーバーからツールを取得するための2つの方式を提供し、さまざまなアプリケーションアーキテクチャに対応します。

| 機能              | 静的構成（`await mcp.getTools()`）            | 動的構成（`await mcp.getToolsets()`）                 |
| :---------------- | :-------------------------------------------- | :--------------------------------------------------- |
| **ユースケース**  | 単一ユーザー向けの静的設定（例: CLI ツール）  | 複数ユーザー向けの動的設定（例: SaaS アプリ）        |
| **構成**          | エージェント初期化時に固定                    | リクエストごとに動的                                 |
| **認証情報**      | すべての利用で共通                            | ユーザー／リクエストごとに個別指定可能               |
| **エージェント設定** | `Agent` コンストラクタでツールを追加          | `.generate()` または `.stream()` のオプションでツールを渡す |

<div id="static-tools">
  ### 静的ツール
</div>

`.getTools()` メソッドを使って、設定済みのすべての MCP サーバーからツールを取得します。これは、設定（API キーなど）がユーザーやリクエスト間で固定されており一貫している場合に適しています。エージェントを定義する際に一度だけ呼び出し、その結果を `tools` プロパティに渡します。

> 詳細は [getTools()](/reference/tools/mcp-client#gettools) を参照してください。

```typescript {8} title="src/mastra/agents/test-agent.ts" showLineNumbers copy
import { openai } from "@ai-sdk/openai";
import { Agent } from "@mastra/core/agent";

import { testMcpClient } from "../mcp/test-mcp-client";

export const testAgent = new Agent({
  // ...
  tools: await testMcpClient.getTools(),
});
```


<div id="dynamic-tools">
  ### 動的ツール
</div>

ツールの構成がリクエストやユーザーごとに変わる場合（各ユーザーが自分の API キーを提供するマルチテナントシステムなど）、`.getToolsets()` メソッドを使用します。このメソッドは、エージェントの `.generate()` や `.stream()` 呼び出しで `toolsets` オプションに渡せるツールセットを返します。

```typescript {5-16,21} showLineNumbers copy
import { MCPClient } from "@mastra/mcp";
import { mastra } from "./mastra";

async function handleRequest(userPrompt: string, userApiKey: string) {
  const userMcp = new MCPClient({
    servers: {
      weather: {
        url: new URL("http://localhost:8080/mcp"),
        requestInit: {
          headers: {
            Authorization: `Bearer ${userApiKey}`,
          },
        },
      },
    },
  });

  const agent = mastra.getAgent("testAgent");

  const response = await agent.generate(userPrompt, {
    toolsets: await userMcp.getToolsets(),
  });

  await userMcp.disconnect();

  return Response.json({
    data: response.text,
  });
}
```

> 詳細は [getToolsets()](/reference/tools/mcp-client#gettoolsets) をご覧ください。


<div id="connecting-to-an-mcp-registry">
  ## MCP レジストリへの接続
</div>

MCP サーバーはレジストリを介して検出できます。`MCPClient` を使って、よく利用されるレジストリに接続する方法は次のとおりです:

<Tabs>
  <TabItem value="klavis" label="Klavis AI">
    [Klavis AI](https://klavis.ai) は、ホスティングされた、エンタープライズ認証対応の高品質な MCP サーバーを提供します。

    ```typescript
    import { MCPClient } from "@mastra/mcp";

    const mcp = new MCPClient({
      servers: {
        salesforce: {
          url: new URL("https://salesforce-mcp-server.klavis.ai/mcp/?instance_id={private-instance-id}"),
        },
        hubspot: {
          url: new URL("https://hubspot-mcp-server.klavis.ai/mcp/?instance_id={private-instance-id}"),
        },
      },
    });
    ```

    Klavis AI は、本番環境向けにエンタープライズ級の認証とセキュリティを提供します。

    Mastra と Klavis の統合方法の詳細は、[ドキュメント](https://docs.klavis.ai/documentation/ai-platform-integration/mastra)をご覧ください。
  </TabItem>

  <TabItem value="mcp-run" label="mcp.run">
    [mcp.run](https://www.mcp.run/) は、事前認証済みのマネージド MCP サーバーを提供します。ツールは「プロファイル」にまとめられ、各プロファイルには固有の署名付き URL が割り当てられます。

    ```typescript
    import { MCPClient } from "@mastra/mcp";

    const mcp = new MCPClient({
      servers: {
        marketing: { // プロファイル名の例
          url: new URL(process.env.MCP_RUN_SSE_URL!), // mcp.runプロファイルからURLを取得
        },
      },
    });
    ```

    > **重要:** mcp.run の SSE URL はパスワード同様に扱ってください。環境変数などに安全に保管してください。
    >
    > ```bash title=".env"
    > MCP_RUN_SSE_URL=https://www.mcp.run/api/mcp/sse?nonce=...
    > ```
  </TabItem>

  <TabItem value="composio" label="Composio.dev">
    [Composio.dev](https://composio.dev) は、[SSE ベースの MCP サーバー](https://mcp.composio.dev) のレジストリを提供しています。Cursor などのツール向けに生成された SSE の URL をそのまま利用できます。

    ```typescript
    import { MCPClient } from "@mastra/mcp";

    const mcp = new MCPClient({
      servers: {
        googleSheets: {
          url: new URL("https://mcp.composio.dev/googlesheets/[非公開URLパス]"),
        },
        gmail: {
          url: new URL("https://mcp.composio.dev/gmail/[非公開URLパス]"),
        },
      },
    });
    ```

    Google Sheets などのサービスの認証は、多くの場合、エージェントとの対話の中でインタラクティブに行われます。

    *注: Composio の URL は通常、単一のユーザーアカウントにひも付くため、マルチテナントアプリケーションよりも個人向けの自動化に適しています。*
  </TabItem>

  <TabItem value="鍛冶場" label="Smithery.ai">
    [Smithery.ai](https://smithery.ai) は、CLI からアクセスできるレジストリを提供しています。

    ```typescript
    // Unix/Mac
    import { MCPClient } from "@mastra/mcp";

    const mcp = new MCPClient({
      servers: {
        sequentialThinking: {
          command: "npx",
          args: [
            "-y",
            "@smithery/cli@latest",
            "run",
            "@smithery-ai/server-sequential-thinking",
            "--config",
            "{}",
          ],
        },
      },
    });
    ```

    ```typescript
    // Windows
    import { MCPClient } from "@mastra/mcp";

    const mcp = new MCPClient({
      servers: {
        sequentialThinking: {
          command: "npx",
          args: [
            "-y",
            "@smithery/cli@latest",
            "run",
            "@smithery-ai/server-sequential-thinking",
            "--config",
            "{}",
          ],
        },
      },
    });
    ```
  </TabItem>

  <TabItem value="アンパサンド（&）" label="アンパサンド記号">
    [Ampersand](https://withampersand.com?utm_source=mastra-docs) は、Salesforce、HubSpot、Zendesk などの SaaS 製品と 150 以上の連携にエージェントを接続できる [MCP Server](https://docs.withampersand.com/mcp) を提供しています。

    ```typescript
    // SSEを使用したAmpersand MCPサーバーを利用するMCPClient
    export const mcp = new MCPClient({
      servers: {
        "@amp-labs/mcp-server": {
          url: `https://mcp.withampersand.com/v1/sse?${new URLSearchParams({
            apiKey: process.env.AMPERSAND_API_KEY,
            project: process.env.AMPERSAND_PROJECT_ID,
            integrationName: process.env.AMPERSAND_INTEGRATION_NAME,
            groupRef: process.env.AMPERSAND_GROUP_REF,
          })}`,
        },
      },
    });
    ```

    ```typescript
    // MCPサーバーをローカルで実行する場合:
    import { MCPClient } from "@mastra/mcp";

    // stdioトランスポートを使用したAmpersand MCP ServerのMCPClient
    export const mcp = new MCPClient({
      servers: {
        "@amp-labs/mcp-server": {
          command: "npx",
          args: [
            "-y",
            "@amp-labs/mcp-server@latest",
            "--transport",
            "stdio",
            "--project",
            process.env.AMPERSAND_PROJECT_ID,
            "--integrationName",
            process.env.AMPERSAND_INTEGRATION_NAME,
            "--groupRef",
            process.env.AMPERSAND_GROUP_REF, // オプション
          ],
          env: {
            AMPERSAND_API_KEY: process.env.AMPERSAND_API_KEY,
          },
        },
      },
    });
    ```

    MCP の代替として、Ampersand の AI SDK には Mastra 用のアダプターも用意されており、エージェントがアクセスできるように [Ampersand のツールを直接インポート](https://docs.withampersand.com/ai-sdk#use-with-mastra) できます。
  </TabItem>
</Tabs>
