<div id="project-structure">
  # プロジェクト構成
</div>

`create mastra` コマンドで作成された新しい Mastra プロジェクトには、すぐに始められるよう、あらかじめ用意されたファイルとフォルダのセットが含まれています。

Mastra はフレームワークですが、ファイルの整理や配置については**干渉しません**。CLI は多くのプロジェクトでうまく機能する妥当なデフォルト構成を提供しますが、ワークフローやチームの慣習に合わせて自由に調整できます。極端な話、望めばプロジェクト全体を1つのファイルで構築することも可能です。どのような構成を選ぶにせよ、一貫性を保つことで、コードの保守性と見通しのよさを確保しましょう。

<div id="default-project-structure">
  ## デフォルトのプロジェクト構成
</div>

`create mastra` コマンドで作成されたプロジェクトの構成は次のとおりです：

```
src/
├── mastra/
│   ├── agents/
│   │   └── weather-agent.ts
│   ├── tools/
│   │   └── weather-tool.ts
│   ├── workflows/
│   │   └── weather-workflow.ts
│   ├── scorers/
│   │   └── weather-scorer.ts
│   └── index.ts
├── .env.example
├── package.json
└── tsconfig.json
```

:::info
ヒント - 既定のファイルをテンプレートとして活用してください。複製して調整すれば、独自のエージェント、ツール、ワークフローなどを素早く作成できます。
:::


<div id="folders">
  ### フォルダー
</div>

フォルダーは、エージェント、ツール、ワークフローなど、エージェントのリソースを整理します。

| Folder                 | Description                                                                                                                              |
| ---------------------- | ---------------------------------------------------------------------------------------------------------------------------------------- |
| `src/mastra`           | Mastra に関するコードと設定のエントリーポイント。                                                                                         |
| `src/mastra/agents`    | エージェントの挙動、目標、使用するツールを定義・設定します。                                                                                |
| `src/mastra/workflows` | エージェントとツールを連携させる複数ステップのワークフローを定義します。                                                                    |
| `src/mastra/tools`     | エージェントから呼び出せる再利用可能なツールを作成します。                                                                                 |
| `src/mastra/mcp`       | （任意）外部のエージェントとツールを共有するためのカスタム MCP サーバーを実装します。                                                      |
| `src/mastra/scorers`   | （任意）時間経過に伴うエージェントのパフォーマンスを評価するスコアラーを定義します。                                                       |
| `src/mastra/public`    | （任意）ビルド時に内容が `.build/output` ディレクトリへコピーされ、実行時に配信可能になります。                                             |

<div id="top-level-files">
  ### トップレベルのファイル
</div>

トップレベルのファイルは、Mastra プロジェクトの設定、ビルド方法、環境との接続を定義します。

| File                  | Description                                                                                                       |
| --------------------- | ----------------------------------------------------------------------------------------------------------------- |
| `src/mastra/index.ts` | Mastra を設定・初期化する中核のエントリーポイント。                                                                |
| `.env.example`        | 環境変数のテンプレート。`.env` にコピーしてリネームし、[model provider](/models) の秘密キーを追加します。            |
| `package.json`        | プロジェクトのメタデータ、依存関係、利用可能な npm スクリプトを定義。                                              |
| `tsconfig.json`       | パスエイリアス、コンパイラ設定、ビルド出力などの TypeScript オプションを設定。                                     |
