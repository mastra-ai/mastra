<div id="contributing-templates">
  # テンプレートへの貢献
</div>

Mastra コミュニティは、革新的なアプリケーション パターンを示すテンプレートの作成において重要な役割を担っています。本ガイドでは、独自のテンプレートを Mastra エコシステムに投稿（貢献）する方法を説明します。

<div id="template-contribution-process">
  ## テンプレートの貢献プロセス
</div>

<div id="1-review-requirements">
  ### 1. 要件の確認
</div>

テンプレートを作成する前に、次の内容を理解していることを確認してください：

- [Templates Reference](/reference/templates/overview) - 技術要件と慣例
- [Project Structure](/docs/getting-started/project-structure) - Mastra プロジェクトの標準的な構成
- コミュニティガイドラインおよび品質基準

<div id="2-develop-your-template">
  ### 2. テンプレートを開発する
</div>

確立されたパターンに沿ってテンプレートを作成します:

- 特定のユースケースまたはパターンにフォーカスする
- 充実したドキュメントを用意する
- クリーンインストールで徹底的にテストする
- すべての技術要件を満たす
- GitHub リポジトリがテンプレートリポジトリになっていることを確認する。[テンプレートリポジトリの作成方法](https://docs.github.com/en/repositories/creating-and-managing-repositories/creating-a-template-repository)

<div id="3-submit-for-review">
  ### 3. レビューに提出する
</div>

テンプレートの準備ができたら、投稿フォームから送信してください。テンプレートは、品質と一貫性を確保するための承認プロセスを経て審査されます。

<div id="submission-guidelines">
  ## 投稿要件
</div>

<div id="template-criteria">
  ### テンプレートの基準
</div>

次のようなテンプレートを受け付けます:

- **独自の価値がある** - 既存のテンプレートではカバーされていない革新的なユースケースやパターンを示している
- **規約に従っている** - すべての技術要件と構造上のガイドラインを満たしている
- **質の高いドキュメントを含む** - 明確なセットアップ手順と使用例がある
- **信頼性高く動作する** - インストール後、最小限の設定で正しく機能する

<div id="quality-standards">
  ### 品質基準
</div>

テンプレートは次の品質基準を満たす必要があります：

- **コード品質** - クリーンで、適切にコメントされ、保守しやすいコード
- **エラー処理** - 外部 API やユーザー入力に対する適切なエラー処理
- **型安全性** - 必要に応じて Zod による検証を伴う、完全な TypeScript の型付け
- **ドキュメント** - セットアップおよび使用方法の手順を含む充実した README
- **テスト** - クリーンな新規インストールでの動作を検証済み

<div id="submission-process">
  ## 提出手順
</div>

<div id="1-prepare-your-template">
  ### 1. テンプレートを準備する
</div>

テンプレートが [Templates Reference](/reference/templates/overview) に記載されたすべての要件を満たしていることを確認してください：

- `src/mastra/` ディレクトリ配下の適切なプロジェクト構成
- 標準的な TypeScript 設定
- 充実した `.env.example` ファイル
- セットアップ手順を詳述した README

<div id="2-submit-your-template">
  ### 2. テンプレートを提出する
</div>

投稿フォームからテンプレートを提出してください:

**[テンプレートの提出](https://forms.gle/g1CGuwFxqbrb3Rz57)**

<div id="required-information">
  ### 必要情報
</div>

テンプレートを提出する際は、次の情報をご提供ください:

- **Template Name** - ユースケースが分かる明確で説明的な名称
- **Template Author Name** - ご自身または組織の名称
- **Template Author Email** - 提出内容に関する連絡用メールアドレス
- **GitHub URL** - テンプレートのリポジトリへのリンク
- **Description** - テンプレートの機能とその価値の詳細な説明
- **Optional Image** - テンプレートの動作を示すスクリーンショットまたは図版
- **Optional Demo Video** - テンプレートの機能をデモする動画へのリンク

<div id="review-process">
  ## レビュー手続き
</div>

<div id="review-criteria">
  ### レビュー基準
</div>

テンプレートは次の観点で評価されます:

- **技術的適合性** - テンプレートのルールや慣習に忠実であること
- **コード品質** - クリーンで保守しやすく、十分にドキュメント化されたコードであること
- **独自性** - 新規性のあるユースケースや革新的な実装パターンであること
- **教育的価値** - Mastra の概念を効果的に学習できるようにする力
- **コミュニティへの貢献** - Mastra コミュニティ全体にとっての有用性

<div id="feedback-and-iteration">
  ### フィードバックと反復
</div>

テンプレートの改善が必要な場合：

- 必要な変更点について具体的なフィードバックが届きます
- 指示された修正を行い、再提出してください
- テンプレートが基準を満たすまで、審査プロセスが継続されます

<div id="community-guidelines">
  ## コミュニティ ガイドライン
</div>

<div id="template-ideas">
  ### テンプレートのアイデア
</div>

次のようなテンプレートの作成を検討してください:

- **業界別ユースケース** - ヘルスケア、金融、教育など
- **インテグレーションパターン** - 特定のAPIやサービスとの連携
- **高度な手法** - 複雑なワークフロー、マルチエージェントシステム、新しいパターン
- **学習リソース** - 特定の概念に関するステップバイステップのチュートリアル

<div id="development-best-practices">
  ### 開発ベストプラクティス
</div>

- **まずはシンプルに** - 動作する最小構成の例から始め、段階的に複雑さを加える
- **十分にドキュメント化する** - 詳細なコメントと充実したREADMEを用意する
- **徹底的にテストする** - さまざまな環境でテンプレートが問題なく動作することを検証する
- **フィードバックを求める** - 提出前にコミュニティで共有し、早期のフィードバックを得る

<div id="community-engagement">
  ### コミュニティへの参加
</div>

- **Discordに参加** - [MastraのDiscordコミュニティ](https://discord.gg/BTYqqHKUrf)に参加しましょう
- **進捗を共有** - テンプレート開発の状況をコミュニティに共有しましょう
- **人を助ける** - 他の貢献者のテンプレート作成を支援しましょう
- **最新情報をチェック** - Mastraの新機能やガイドラインを把握しましょう

<div id="template-maintenance">
  ## テンプレートの保守
</div>

<div id="ongoing-responsibilities">
  ### 継続的な責務
</div>

テンプレートのコントリビューターとして、以下をお願いする場合があります:

- **依存関係の更新** - テンプレートを最新の Mastra に対応させ、常に最新状態に保つ
- **問題の修正** - バグや互換性の問題に対応する
- **ドキュメントの改善** - ユーザーからのフィードバックに基づき手順を充実させる
- **機能の追加** - 新機能でテンプレートを拡張する

<div id="community-support">
  ### コミュニティサポート
</div>

Mastra のチームとコミュニティは以下を提供します：

- **技術支援** - 複雑な実装課題へのサポート
- **レビューフィードバック** - テンプレート品質を高めるための詳細なフィードバック
- **プロモーション** - 承認済みテンプレートのコミュニティでの紹介
- **保守支援** - テンプレートを最新状態に保つためのサポート

<div id="validation-checklist">
  ## 検証チェックリスト
</div>

テンプレートを提出する前に、次を確認してください:

- [ ] すべてのコードが `src/mastra/` ディレクトリに整理されている
- [ ] 標準の Mastra の TypeScript 設定を使用している
- [ ] 充実した `.env.example` が含まれている
- [ ] セットアップ手順を含む詳細な README がある
- [ ] モノレポや Web フレームワークのボイラープレートがない
- [ ] クリーンインストールと環境構築後に問題なく起動・実行できる
- [ ] すべてのコード品質基準に準拠している
- [ ] 明確で有用なユースケースが示されている

  ### 表彰
</div>

テンプレートの貢献者には以下が提供されます：

- **クレジット表記** - テンプレートにあなたの名前と連絡先情報を掲載
- **コミュニティでの認知** - コミュニティチャンネルでの謝意の表明

<div id="getting-started">
  ## はじめに
</div>

テンプレートの投稿を始める準備はできましたか？

1. **既存のテンプレートをチェック** - 現在のテンプレートを見て、発想やパターンの参考にしましょう
2. **テンプレートを設計** - ユースケースと提供価値を明確にしましょう
3. **要件を順守** - すべての技術要件に適合していることを確認しましょう
4. **作成とテスト** - 動作し、十分にドキュメント化されたテンプレートを作りましょう
5. **レビューに提出** - 投稿フォームからテンプレートを提出しましょう

:::info

皆さんの貢献はMastraエコシステムの拡大に寄与し、コミュニティ全体にとって価値あるリソースとなります。皆さんの革新的なテンプレートを楽しみにしています！

:::
