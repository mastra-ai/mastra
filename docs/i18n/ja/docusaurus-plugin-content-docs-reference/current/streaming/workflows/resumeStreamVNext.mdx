<div id="runresumestreamvnext-experimental">
  # Run.resumeStreamVNext()（実験的）
</div>

:::caution 実験的機能
これは将来のバージョンで変更される可能性のある実験的な API です。`resumeStreamVNext()` メソッドは、最終的に現在のストリーミング手法を置き換える強化版ストリーミングシステムの一部です。本番環境での使用は慎重に検討してください。
:::

`.resumeStreamVNext()` メソッドは、新しいデータを用いて一時停止中のワークフロー実行を再開し、特定のステップから続行すると同時に、イベントのストリームを監視できるようにします。

<div id="usage-example">
  ## 使い方の例
</div>

```typescript showLineNumbers copy
const run = await workflow.createRunAsync();

const stream = run.streamVNext({
  inputData: {
    value: "初期データ",
  },
});

const result = await stream.result;

if (result!.status === "suspended") {
  const resumedStream = await run.resumeStreamVNext({
    resumeData: {
      value: "再開データ",
    },
  });
}
```

<div id="stream-events">
  ## ストリームイベント
</div>

ストリームはワークフローの実行中にさまざまなイベントを送出します。各イベントには `type` フィールドと、関連するデータを含む `payload` が含まれます:

- **`workflow-start`**: ワークフローの実行が開始される
- **`workflow-step-start`**: ステップの実行が開始される
- **`workflow-step-output`**: ステップからのカスタム出力
- **`workflow-step-result`**: ステップが結果とともに完了する
- **`workflow-finish`**: 使用状況の統計とともにワークフローの実行が完了する
