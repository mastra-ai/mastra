# Run.observeStreamVNext()（実験的）

`.observeStreamVNext()` メソッドは、現在進行中のワークフロー実行に対して新しい `ReadableStream` を開き、元のストリームが利用できなくなった場合にイベントのストリームを引き続き監視できるようにします。

## 使い方の例

```typescript showLineNumbers copy
const run = await workflow.createRunAsync();

run.streamVNext({
  inputData: {
    value: "初期データ",
  },
});

const stream = await run.observeStreamVNext();

for await (const chunk of stream) {
  console.log(chunk);
}
```


## 戻り値

`ReadableStream<ChunkType>`

## ストリームイベント

ストリームはワークフローの実行中にさまざまな種類のイベントを発行します。各イベントには `type` フィールドと、関連データを含む `payload` が含まれます:

- **`workflow-start`**: ワークフローの実行を開始
- **`workflow-step-start`**: ステップの実行を開始
- **`workflow-step-output`**: ステップからのカスタム出力
- **`workflow-step-result`**: ステップが結果とともに完了
- **`workflow-finish`**: 使用状況の統計とともにワークフローの実行が完了
