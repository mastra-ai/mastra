<div id="runresume">
  # run.resume()
</div>

`.resume()` メソッドは、一時停止中のワークフローステップの実行を再開し、必要に応じてステップが `inputData` プロパティから参照できる新しいコンテキストデータを渡せます。

<div id="usage">
  ## 使い方
</div>

```typescript copy showLineNumbers
await run.resume({
  runId: "abc-123",
  stepId: "stepTwo",
  context: {
    secondValue: 100,
  },
});
```

<div id="asyncawait-flow">
  ## Async/Await のフロー
</div>

ワークフローが再開されると、処理はステップの実行関数内での `suspend()` 呼び出し直後の位置から再開されます。これにより、コードは自然な流れになります。

```typescript
// サスペンドポイントを含むステップ定義
const reviewStep = new LegacyStep({
  id: "review",
  execute: async ({ context, suspend }) => {
    // 実行の前半部分
    const initialAnalysis = analyzeData(context.inputData.data);

    if (initialAnalysis.needsReview) {
      // ここで実行をサスペンドします
      await suspend({ analysis: initialAnalysis });

      // このコードは resume() が呼び出された後に実行されます
      // context.inputData には、resume 時に渡されたデータが含まれます
      return {
        reviewedData: enhanceWithFeedback(
          initialAnalysis,
          context.inputData.feedback,
        ),
      };
    }

    return { reviewedData: initialAnalysis };
  },
});

const { runId, resume, start } = workflow.createRun();

await start({
  inputData: {
    data: "some data",
  },
});

// 後でワークフローを再開します
const result = await resume({
  runId: "workflow-123",
  stepId: "review",
  context: {
    // このデータは `context.inputData` から参照できます
    feedback: "良さそうですが、セクション3を改善してください",
  },
});
```


<div id="execution-flow">
  ### 実行フロー
</div>

1. ワークフローは `review` ステップ内の `await suspend()` に到達するまで実行されます
2. ワークフローの状態が永続化され、実行が一時停止します
3. 後で、新しいコンテキストデータとともに `run.resume()` が呼び出されます
4. 実行は `review` ステップ内の `suspend()` の直後から再開されます
5. 新しいコンテキストデータ（`feedback`）は `inputData` プロパティ経由で当該ステップから利用できます
6. ステップが完了し、結果を返します
7. ワークフローは後続のステップへと進みます

<div id="error-handling">
  ## エラー処理
</div>

resume 関数はいくつかの種類のエラーをスローする可能性があります：

```typescript
try {
  await run.resume({
    runId,
    stepId: "stepTwo",
    context: newData,
  });
} catch (error) {
  if (error.message === "ワークフロー実行のスナップショットが見つかりません") {
    // ワークフローの状態が見つからない場合の処理
  }
  if (error.message === "ワークフローのスナップショットを解析できませんでした") {
    // 破損したワークフロー状態の処理
  }
}
```
