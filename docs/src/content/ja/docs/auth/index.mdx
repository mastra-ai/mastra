# 認証の概要

Mastra では認証方式を自由に選べるため、スタックに適したアイデンティティシステムでアプリケーションのエンドポイントへのアクセスを保護できます。

まずはシンプルな共有シークレットを用いた JWT 認証から始め、より高度なアイデンティティ機能が必要になったら、Supabase、Firebase Auth、Auth0、Clerk、WorkOS などのプロバイダーへ切り替えることができます。

## 利用できるプロバイダー

- [JSON Web Token (JWT)](/docs/auth/jwt)
- [Clerk](/docs/auth/clerk)
- [Supabase](/docs/auth/supabase)
- [Firebase](/docs/auth/firebase)
- [WorkOS](/docs/auth/workos)
