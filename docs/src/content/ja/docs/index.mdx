# Mastra について

Gatsby を手がけたチームによる Mastra は、モダンな TypeScript スタックで AI 搭載のアプリケーションやエージェントを構築するためのフレームワークです。

試作段階から本番運用レベルのアプリケーションまでに必要なものがすべて揃っています。Mastra は React、Next.js、Node などのフロントエンド／バックエンドフレームワークと統合でき、単体のサーバーとしてどこにでもデプロイすることも可能です。信頼性の高い AI プロダクトを構築・チューニング・スケールする最も簡単な方法です。

## なぜ Mastra なのか？

TypeScript 向けに特化設計され、確立された AI パターンに基づく Mastra は、優れた AI アプリケーションをすぐに構築できるよう、必要な機能をすべて提供します。

主なポイント:

- [**モデルルーティング**](/models) - 1 つの標準インターフェースで 40 社以上のプロバイダーに接続。OpenAI、Anthropic、Gemini などのモデルを利用できます。

- [**エージェント**](/docs/agents/overview) - LLM とツールを活用してオープンエンドなタスクを解決する自律エージェントを構築。エージェントは目標に基づいて推論し、使うツールを選択し、モデルが最終回答を出すか、任意の停止条件を満たすまで内部で反復します。

- [**ワークフロー**](/docs/workflows/overview) - 実行を明示的に制御したい場合は、Mastra のグラフベースのワークフローエンジンで複雑な多段処理をオーケストレーション。Mastra のワークフローは制御フローに直感的な構文（`.then()`、`.branch()`、`.parallel()`）を採用しています。

- [**Human-in-the-loop**](/docs/workflows/suspend-and-resume) - エージェントやワークフローを一時停止し、再開前にユーザー入力や承認を待機可能。Mastra は[ストレージ](/docs/server-db/storage)で実行状態を保持するため、無期限に一時停止し、中断地点から再開できます。

- **コンテキスト管理** - 適切なタイミングで適切なコンテキストをエージェントに付与。[会話履歴](/docs/memory/conversation-history)を提供し、ソース（API、データベース、ファイル）からデータを[検索・取得](/docs/rag/overview)し、人間らしい[ワーキング](/docs/memory/working-memory)および[セマンティック](/docs/memory/semantic-recall)メモリを付与して、一貫した振る舞いを実現します。 

- **インテグレーション** - エージェントやワークフローを既存の React、Next.js、Node.js アプリに組み込むか、スタンドアロンのエンドポイントとして提供。UI 構築時は、Vercel の AI SDK UI や CopilotKit などのエージェント指向ライブラリと統合して、ウェブ上で AI アシスタントを実装できます。

- **本番運用の必携機能** - 信頼性の高いエージェントを提供するには、継続的なインサイト、評価、反復が不可欠。組み込みの[評価](/docs/evals/overview)と[可観測性](/docs/observability/overview)により、Mastra は観測・計測・継続的な改善のためのツールを提供します。

## 何が作れる？

- 言語理解・推論・アクションを組み合わせ、現実の課題を解決するAI搭載アプリケーション。

- カスタマーサポート、オンボーディング、社内問い合わせ向けの会話型エージェント。

- コーディング、法務、財務、リサーチ、クリエイティブワークといった分野に特化したコパイロット。

- 発火・ルーティングから完了までを自動化するマルチステップのワークフロー。

- データを分析し、実行可能な提言を行う意思決定支援ツール。

実例は[ケーススタディ](/blog/category/case-studies)や[コミュニティショーケース](/showcase)をご覧ください。
