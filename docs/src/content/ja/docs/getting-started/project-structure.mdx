# プロジェクト構成

`create mastra` コマンドで作成された新しい Mastra プロジェクトには、すぐに始められるよう、あらかじめ用意されたファイルとフォルダのセットが含まれています。

Mastra はフレームワークですが、ファイルの整理や配置については**干渉しません**。CLI は多くのプロジェクトでうまく機能する妥当なデフォルト構成を提供しますが、ワークフローやチームの慣習に合わせて自由に調整できます。極端な話、望めばプロジェクト全体を1つのファイルで構築することも可能です。どのような構成を選ぶにせよ、一貫性を保つことで、コードの保守性と見通しのよさを確保しましょう。

## デフォルトのプロジェクト構成

`create mastra` コマンドで作成されたプロジェクトは、次のような構成になります:

</Callout>

### フォルダ

フォルダは、エージェント、ツール、ワークフローなど、エージェントのリソースを整理します。

| フォルダ                 | 説明 |
| ---------------------- | ------------ |
| `src/mastra`           | Mastra 関連のコードと設定のすべてのエントリーポイント。|
| `src/mastra/agents`    | エージェントの挙動、目標、ツールを定義・設定します。 |
| `src/mastra/workflows` | エージェントとツールを連携させてオーケストレーションする複数ステップのワークフローを定義します。 |
| `src/mastra/tools`     | エージェントが呼び出せる再利用可能なツールを作成します。 |
| `src/mastra/mcp`       | （任意）外部のエージェントとツールを共有するためのカスタム MCP サーバーを実装します。 |
| `src/mastra/scorers`   | （任意）経時的にエージェントのパフォーマンスを評価するためのスコアラーを定義します。 |
| `src/mastra/public`    | （任意）内容はビルド時に `.build/output` ディレクトリへコピーされ、実行時に配信可能になります。 |

### トップレベルのファイル

トップレベルのファイルは、Mastra プロジェクトの設定、ビルド、そして実行環境との連携方法を定義します。

| File                  | Description |
| --------------------- | ------------ |
| `src/mastra/index.ts` | Mastra を設定・初期化する中核となるエントリーポイント。 |
| `.env.example`        | 環境変数のテンプレート — `.env` にコピーしてリネームし、[モデルプロバイダー](/models) のシークレットキーを追加します。 |
| `package.json`        | プロジェクトのメタデータ、依存関係、利用可能な npm スクリプトを定義します。 |
| `tsconfig.json`       | パスエイリアス、コンパイラ設定、ビルド出力などの TypeScript オプションを設定します。 |
