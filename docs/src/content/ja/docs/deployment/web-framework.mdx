# Web Framework Integration

このガイドでは、Mastra を統合したアプリケーションのデプロイ方法を説明します。Mastra はさまざまな Web フレームワークと連携でき、詳細は次のいずれかのガイドをご覧ください。

- [Next.js との連携](/docs/frameworks/web-frameworks/next-js)
- [Astro との連携](/docs/frameworks/web-frameworks/astro)

フレームワークと連携している場合、Mastra は通常、デプロイに追加の設定は不要です。

## Vercel 上の Next.js で

Mastra を Next.js と統合している場合は、[こちらのガイド](/docs/frameworks/web-frameworks/next-js)に従っていれば、Vercel へのデプロイに追加のセットアップは不要です。

確認すべきなのは、`next.config.ts` に以下を追加し、サーバーレス環境ではサポートされない [LibSQLStore](/docs/deployment/deployment#libsqlstore) の使用を削除していることだけです。

```typescript {4} filename="next.config.ts" showLineNumbers copy
import type { NextConfig } from "next";

const nextConfig: NextConfig = {
  serverExternalPackages: ["@mastra/*"],
};

export default nextConfig;
```


## Vercel 上での Astro

ガイドに沿って Mastra を Astro と[統合し](/docs/frameworks/web-frameworks/astro)、Vercel へデプロイする予定であれば、追加の設定は不要です。

確認すべき点はただ一つ、`astro.config.mjs` に以下を追加し、サーバーレス環境ではサポートされない [LibSQLStore](/docs/deployment/deployment#libsqlstore) の使用を削除していることです。

```javascript {2,6,7} filename="astro.config.mjs" showLineNumbers copy
import { defineConfig } from 'astro/config';
import vercel from '@astrojs/vercel';

export default defineConfig({
  // 省略
  adapter: vercel(),
  output: "server"
});
```


## Netlify での Astro

Mastra を Astro と[ガイドに従って](/docs/frameworks/web-frameworks/astro)統合し、Vercel にデプロイする予定であれば、追加の設定は不要です。

確認すべき点はただひとつ、`astro.config.mjs` に以下を追加し、サーバーレス環境ではサポートされない [LibSQLStore](/docs/deployment/deployment#libsqlstore) の使用を削除していることです。

```javascript {2,6,7} filename="astro.config.mjs" showLineNumbers copy
import { defineConfig } from 'astro/config';
import vercel from '@astrojs/netlify';

export default defineConfig({
  // 省略
  adapter: netlify(),
  output: "server"
});
```
