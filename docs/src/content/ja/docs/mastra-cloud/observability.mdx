# トレーシングとログを理解する

Mastra Cloud は、本番環境でのアプリケーションの挙動を監視できるよう、実行データを取得します。

<MastraCloudCallout />

## ログ

Dashboard の [Logs](/docs/mastra-cloud/dashboard#logs) ページで、アプリケーションの挙動をデバッグ・監視するための詳細なログを確認できます。

![Dashboard logs](/image/mastra-cloud/mastra-cloud-dashboard-logs.jpg)

主な機能:

各ログエントリには、重大度レベルと、エージェント、ワークフロー、ストレージの各アクティビティを示す詳細なメッセージが含まれます。

## トレース

より詳細なトレースは、[logger](/docs/observability/logging) を使用するか、[サポートされているプロバイダー](/reference/observability/providers) のいずれかで [telemetry](/docs/observability/tracing) を有効化することで、エージェントとワークフローの双方で利用できます。

### エージェント

[logger](/docs/observability/logging) を有効にすると、Agents Playground の **Traces** セクションでエージェントの詳細な出力を確認できます。

![observability agents](/image/mastra-cloud/mastra-cloud-observability-agents.jpg)

主な機能:

生成中にエージェントに渡されるツールは、`convertTools` を用いて標準化されます。これには、クライアント側のツール、メモリツール、ワークフローから公開されるツールの取得が含まれます。

### ワークフロー

[logger](/docs/observability/logging) を有効にすると、Workflows Playground の **Traces** セクションでワークフローの詳細な出力を確認できます。

![observability workflows](/image/mastra-cloud/mastra-cloud-observability-workflows.jpg)

主な機能:

ワークフローは `createWorkflow` を使って作成し、ステップ、メタデータ、ツールを設定します。入力とオプションを渡して `runWorkflow` で実行できます。
