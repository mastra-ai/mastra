# Workflow.else()

> 実験的

`.else()` メソッドは、直前の `if` 条件が false と評価された場合に実行される代替ブランチをワークフロー内に作成します。これにより、条件に応じてワークフローが異なるパスをたどれるようになります。

## 使い方

```typescript copy showLineNumbers
workflow
  .step(startStep)
  .if(async ({ context }) => {
    const value = context.getStepResult<{ value: number }>("start")?.value;
    return value < 10;
  })
  .then(ifBranchStep)
  .else() // 条件が偽のときの代替分岐
  .then(elseBranchStep)
  .commit();
```


## パラメータ

`else()` メソッドには引数はありません。

## 挙動

- `else()` メソッドは、ワークフロー定義内で `if()` 分岐の直後に続けて記述する必要があります
- 直前の `if` 条件が false と評価された場合にのみ実行される分岐を作成します
- `.then()` を使って `else()` の後に複数のステップをチェーンできます
- `else` 分岐内に追加の `if`/`else` 条件をネストできます

## エラーハンドリング

`else()` メソッドを使うには、先行する `if()` ステートメントが必要です。先行する `if` なしで使用すると、エラーが発生します。

```typescript
try {
  // これはエラーを投げます
  workflow.step(someStep).else().then(anotherStep).commit();
} catch (error) {
  console.error(error); // 「有効な条件が見つかりません」
}
```
