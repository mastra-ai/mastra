# voice.addInstructions()

`addInstructions()` メソッドは、リアルタイムのやり取りにおけるモデルの振る舞いを導く指示を音声プロバイダーに付与します。これは、会話全体でコンテキストを保持するリアルタイム音声プロバイダーに特に有用です。

## 使い方の例

```typescript
import { OpenAIRealtimeVoice } from "@mastra/voice-openai-realtime";
import { Agent } from "@mastra/core/agent";
import { openai } from "@ai-sdk/openai";

// リアルタイム音声プロバイダーを初期化する
const voice = new OpenAIRealtimeVoice({
  realtimeConfig: {
    model: "gpt-4o-mini-realtime",
    apiKey: process.env.OPENAI_API_KEY,
  },
});

// 音声プロバイダーを使ってエージェントを作成する
const agent = new Agent({
  name: "カスタマーサポート担当",
  instructions:
    "あなたはソフトウェア企業の親切なカスタマーサポート担当者です。",
  model: openai("gpt-4o"),
  voice,
});

// 音声プロバイダーに追加の指示を設定する
voice.addInstructions(`
  お客様とお話しする際は、以下に留意してください：
  - つねにカスタマーサポート担当者として名乗る
  - 明瞭かつ簡潔に話す
  - 必要に応じて確認のための質問をする
  - 会話の最後に内容を要約する
`);

// リアルタイムサービスに接続する
await voice.connect();
```

## 戻り値

このメソッドは値を返しません。

## 注意事項

- 指示は、明確で具体的で、音声でのやり取りに関係しているほど効果的です
- このメソッドは、会話コンテキストを維持するリアルタイム音声プロバイダーで主に使用されます
- 指示をサポートしない音声プロバイダーで呼び出された場合は、警告をログに記録し、何もしません
- このメソッドで追加した指示は、関連する Agent が提供する指示と通常は統合されます
- 最良の結果を得るには、会話を開始する前（`connect()` を呼び出す前）に指示を追加してください
- `addInstructions()` を複数回呼び出すと、プロバイダーの実装によっては、既存の指示を置き換えるか、追記されます
