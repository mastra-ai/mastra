Azure OpenAI Service provides access to OpenAI's models through Microsoft Azure infrastructure. Unlike the standard OpenAI provider, Azure OpenAI uses **deployment-based model names** that you configure in the Azure Portal.

## Prerequisites

Before using Azure OpenAI with Mastra, you need to:

1. **Create an Azure OpenAI resource** in the [Azure Portal](https://portal.azure.com)
2. **Deploy a model** to your Azure OpenAI resource (e.g., gpt-4, gpt-35-turbo)
3. **Note your deployment name** - this is what you'll use in Mastra, not the underlying model name

## Environment Variables

```bash
# Required
AZURE_RESOURCE_NAME=your-resource-name  # From Azure Portal (e.g., "my-openai-resource")
AZURE_API_KEY=your-api-key              # From Azure Portal "Keys and Endpoint"

# Optional
OPENAI_API_VERSION=2024-10-21          # Defaults to 2024-10-21 if not specified
```

### Understanding AZURE_RESOURCE_NAME

The `AZURE_RESOURCE_NAME` is the name of your Azure OpenAI resource as it appears in the Azure Portal. For example, if your Azure OpenAI endpoint is:

```
https://my-openai-resource.openai.azure.com/
```

Then your `AZURE_RESOURCE_NAME` should be:

```bash
AZURE_RESOURCE_NAME=my-openai-resource
```

**Important:** Use only the resource name, not the full URL.

## Model ID Format

Azure OpenAI uses a unique model ID format in Mastra:

```typescript
model: "azure/<deployment-name>"
```

The `<deployment-name>` is the name you assigned when deploying the model in Azure Portal, **not** the underlying OpenAI model name.

### Example

If you deployed `gpt-4` in Azure Portal and named the deployment `my-gpt4-deployment`, you would use:

```typescript
const agent = new Agent({
  id: "my-agent",
  name: "My Agent",
  instructions: "You are a helpful assistant",
  model: "azure/my-gpt4-deployment"
});
```

## Basic Usage

```typescript
import { Agent } from "@mastra/core/agent";

const agent = new Agent({
  id: "azure-agent",
  name: "Azure Agent",
  instructions: "You are a helpful assistant",
  model: "azure/my-gpt4-deployment"  // Use your Azure deployment name
});

// Generate a response
const response = await agent.generate("Hello!");
console.log(response.text);

// Stream a response
const stream = await agent.stream("Tell me a story");
for await (const chunk of stream) {
  console.log(chunk);
}
```

## Important Notes

### Deployment Names vs Model Names

- **Deployment Name**: The name you choose in Azure Portal (e.g., "my-gpt4-deployment", "production-gpt35")
- **Model Name**: The underlying OpenAI model (e.g., "gpt-4", "gpt-35-turbo")

Mastra uses the **deployment name**, which can be different from the underlying model name. This allows you to:
- Use descriptive names that match your deployment strategy
- Have multiple deployments of the same model with different configurations
- Easily switch between development and production deployments

### API Versions

Azure OpenAI API versions evolve over time. The default version (`2024-10-21`) is set automatically, but you can override it:

```bash
OPENAI_API_VERSION=2025-04-01-preview
```

Check the [Azure OpenAI API versions documentation](https://learn.microsoft.com/en-us/azure/ai-services/openai/api-version-deprecation) for the latest available versions.

## Advanced Configuration

### Custom Headers

```typescript
const agent = new Agent({
  id: "custom-agent",
  name: "Custom Agent",
  model: {
    id: "azure/my-deployment",
    apiKey: process.env.AZURE_API_KEY,
    headers: {
      "X-Custom-Header": "value"
    }
  }
});
```

### Dynamic Model Selection

```typescript
const agent = new Agent({
  id: "dynamic-agent",
  name: "Dynamic Agent",
  model: ({ requestContext }) => {
    const useAdvanced = requestContext.task === "complex";
    return useAdvanced
      ? "azure/gpt4-deployment"
      : "azure/gpt35-deployment";
  }
});
```

## Troubleshooting

### "AZURE_RESOURCE_NAME environment variable is required"

Make sure you've set the `AZURE_RESOURCE_NAME` environment variable to your Azure OpenAI resource name (not the full URL).

```bash
# Correct
AZURE_RESOURCE_NAME=my-openai-resource

# Incorrect
AZURE_RESOURCE_NAME=https://my-openai-resource.openai.azure.com/
```

### "Could not find API key process.env.AZURE_API_KEY"

Ensure the `AZURE_API_KEY` environment variable is set with your Azure OpenAI API key from the Azure Portal.

### "Deployment not found"

Verify that:
1. The deployment name in your model ID matches exactly what's in Azure Portal
2. The deployment is in an "Active" state in Azure Portal
3. You're using the correct Azure resource name

### Connection Issues

If you experience connection issues:
1. Verify your Azure OpenAI resource is in an active region
2. Check that your API key is valid and hasn't expired
3. Ensure your Azure subscription is active
4. Review Azure service health status

## Common Deployment Patterns

While you can name your Azure deployments anything, here are common patterns:

```typescript
// Development
model: "azure/dev-gpt4"
model: "azure/dev-gpt35-turbo"

// Production
model: "azure/prod-gpt4"
model: "azure/prod-gpt35-turbo"

// By use case
model: "azure/customer-support-gpt4"
model: "azure/content-generation-gpt35"
```
