# Azure OpenAI

Azure OpenAI Service provides access to OpenAI models through Microsoft Azure infrastructure.

## Prerequisites

1. **Azure OpenAI Resource:** Create an Azure OpenAI resource in the Azure Portal
2. **Model Deployment:** Deploy a model (e.g., GPT-4) in your Azure resource
3. **Deployment Name:** Note your deployment name (you'll use this in model IDs)

## Environment Variables

Set these environment variables:

```bash
# Required
AZURE_RESOURCE_NAME=your-resource-name  # Just the name, not the full URL
AZURE_API_KEY=your-api-key

# Optional (defaults to '2024-10-21')
OPENAI_API_VERSION=2024-10-21
```

## Model ID Format

Use `azure/<deployment-name>` format:

```typescript
import { Agent } from '@mastra/core';

const agent = new Agent({
  model: 'azure/gpt-4.1-mini',  // Your deployment name
  // ... other config
});
```

## Important Notes

### Deployment Names
- The model ID must match your **deployment name** in Azure, not the model name
- Example: If you named your deployment "my-gpt4", use `azure/my-gpt4`
- Common pattern: Deploy with names like `gpt-4.1-mini`, `gpt-4o`, etc.

### Resource Name
- Use just the resource name, not the full URL
- Example: If your endpoint is `https://my-resource.openai.azure.com`, use `AZURE_RESOURCE_NAME=my-resource`

### API Versions
- Defaults to `2024-10-21` (stable)
- Override with `OPENAI_API_VERSION` env var
- See [Azure OpenAI API versions](https://learn.microsoft.com/azure/ai-services/openai/api-version-deprecation)

## Example

```typescript
// Set environment variables
process.env.AZURE_RESOURCE_NAME = 'my-resource';
process.env.AZURE_API_KEY = 'abc123...';
process.env.OPENAI_API_VERSION = '2024-10-21';

// Create agent with Azure model
const agent = new Agent({
  model: 'azure/gpt-4.1-mini',
  instructions: 'You are a helpful assistant',
});

// Generate response
const response = await agent.generate('Hello!');
```

## Common Models

Assuming you've deployed these models with matching names:

- `azure/gpt-4` - GPT-4
- `azure/gpt-4.1` - GPT-4 Turbo
- `azure/gpt-4.1-mini` - GPT-4 Mini
- `azure/gpt-4o` - GPT-4o
- `azure/o1` - o1
- `azure/o1-mini` - o1 Mini
- `azure/o1-preview` - o1 Preview

## Troubleshooting

### "AZURE_RESOURCE_NAME environment variable is required"
- Ensure `AZURE_RESOURCE_NAME` is set in your environment
- Use just the resource name, not the full URL

### "DeploymentNotFound" error from Azure
- Your deployment name in Azure doesn't match the model ID
- Check your deployment name in Azure Portal
- Use `azure/<your-actual-deployment-name>`

### Authentication errors
- Verify `AZURE_API_KEY` is correct
- Check key hasn't expired in Azure Portal
- Ensure resource has proper permissions
