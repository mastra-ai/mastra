# Modes

Modes define the different personalities or capability sets available within a Harness. Each mode has its own agent (or agent factory), optional tool overrides, and metadata. The Harness ensures only one mode is active at a time and handles transitions between them.

## When to use modes

Use modes when a single conversational session needs to switch between distinct behaviors — for example, a planning mode that reasons about tasks and a build mode that executes them. Modes share the same thread and state, so the agent retains context across switches.

## Quickstart

```typescript title="src/mastra/harness.ts"
import { Harness } from '@mastra/core/harness'
import { planAgent, buildAgent } from './agents'

const harness = new Harness({
  id: 'multi-mode',
  modes: [
    { id: 'plan', name: 'Plan', metadata: { default: true }, agent: planAgent },
    { id: 'build', name: 'Build', agent: buildAgent },
  ],
})

await harness.init()
```

## Defining modes

Each mode requires an `id` and an `agent`. The agent can be a static `Agent` instance or a factory function that receives the current harness state:

```typescript title="src/mastra/harness.ts"
import { Harness } from '@mastra/core/harness'

const harness = new Harness({
  id: 'dynamic-modes',
  modes: [
    {
      id: 'plan',
      name: 'Plan',
      metadata: { default: true },
      defaultModelId: '__GATEWAY_ANTHROPIC_MODEL_SONNET__',
      agent: planAgent,
    },
    {
      id: 'build',
      name: 'Build',
      defaultModelId: '__GATEWAY_ANTHROPIC_MODEL_SONNET__',
      agent: buildAgent,
    },
  ],
})
```

### Mode tool overrides

Modes support two strategies for tool configuration. Use `tools` to replace the backing agent's tools entirely, or `additionalTools` to layer extra tools on top:

```typescript
// Replace — agent sees only these tools in plan mode
const planMode = { id: 'plan', agent: baseAgent, tools: { planTool } }

// Augment — agent keeps its own tools plus these
const buildMode = { id: 'build', agent: baseAgent, additionalTools: { deployTool } }
```

You cannot set both `tools` and `additionalTools` on the same mode.

### Mode transitions

A mode can declare a `transitionsTo` target. When the `submit_plan` built-in tool runs in that mode, the harness transitions to the target mode on approval:

```typescript
const planMode = {
  id: 'plan',
  name: 'Plan',
  transitionsTo: 'build',
  agent: planAgent,
}
```

On plan approval, the harness automatically switches to `build` mode. On rejection, the agent remains in `plan` mode to revise.

## Switching modes

Call `switchMode` to change the active mode. The harness aborts any in-progress generation, saves the current model to the outgoing mode, loads the incoming mode's model, and emits `mode_changed` and `model_changed` events:

```typescript
await harness.switchMode({ modeId: 'build' })
```

## Querying modes

```typescript
// List all configured modes
const modes = harness.listModes()

// Get the current mode ID
const modeId = harness.getCurrentModeId()

// Get the full mode object
const mode = harness.getCurrentMode()
```
