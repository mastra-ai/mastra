# Studio desktop app

Mastra Studio Desktop is an installable Electron shell for Studio. It opens Studio tabs for a bundled local runtime, a running Mastra dev server, or a hosted Studio project from a Platform account.

## When to use the desktop app

Use the desktop app when you want Studio as a desktop control surface for local and hosted Mastra projects.

- Local model workflows: Connect Studio to LM Studio through `http://localhost:1234/v1`.
- Local development: Connect a tab to a running `mastra dev` server, such as `http://127.0.0.1:4111`.
- Hosted project access: Sign in to Mastra Platform and open deployed Studio projects from the app.
- Local demos: Use the bundled starter runtime, Studio UI, settings, and logs in one app window.

For custom Electron applications that call a Mastra server from their own UI, use the [Electron framework guide](/guides/getting-started/electron).

## Studio sources

The desktop app opens a new tab picker when you create a tab. The picker supports these Studio sources:

- **Bundled Template**: Starts the packaged Mastra runtime and opens Studio against it.
- **Localhost :4111**: Probes the default Mastra dev server URL and opens Studio when `/api/agents` responds.
- **Connect local server**: Accepts a port or URL for another running Mastra dev server.
- **Platform Studios**: Lists hosted Studio projects after you sign in to Mastra Platform.

Platform Studio tabs use the same deploy single sign-on URL as the Platform web app. The desktop app lists and opens hosted Studios, but it doesn't create deployments or manage cloud runtime state in this version.

## Local model preset

The desktop app ships with an LM Studio preset that targets `http://localhost:1234/v1`. Use **Model > Check LM Studio Server** to query `http://localhost:1234/v1/models` and list any model IDs returned by the local server.

Use **Model > Use LM Studio Preset** to switch the managed runtime to LM Studio settings. When LM Studio reports loaded models, the app uses the first returned model ID. Otherwise, it keeps the default local model ID and lets you update the model in the app settings file.

If LM Studio isn't running, start the LM Studio local server and run the check again. The desktop app doesn't start or manage LM Studio in this version.

## Data and runtime

In managed mode, the app starts a bundled Mastra runtime bound to `127.0.0.1`. It stores settings and local Editor files in the app data directory for the current operating system.

The bundled starter runtime includes:

- a `desktopAssistant` agent
- local filesystem storage for Editor drafts and versioning
- `@mastra/editor`
- model configuration from the desktop settings
