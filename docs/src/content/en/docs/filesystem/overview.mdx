# Filesystem overview

:::experimental

This feature is in beta. Breaking changes may occur without a major version bump until the API is stable.

:::

With filesystem routing, the directory is the agent. Each folder under `src/mastra/agents/` becomes an [`Agent`](/reference/agents/agent) instance, its directory name becomes the default `id` and `name`, and each file inside the folder adds one capability.

A file-based project starts with `config.ts` and `instructions.md`. Add more files only when the agent needs more capabilities.

## Grow by adding files

File-based agents use a default-first model. Start with the files required to run, then add folders as the agent grows.

| File or folder | What it adds |
| - | - |
| `config.ts` | Model selection and runtime options |
| `instructions.md` | The always-on system prompt |
| `tools/` | Typed functions the model can call |
| `skills/` | Procedures loaded on demand |
| `memory.ts` | Persistent conversation memory |
| `workspace.ts` | A filesystem and sandbox |
| `workspace/` | Seed files shipped with the agent |
| `processors/` | Input/output processing pipeline |
| `subagents/` | Specialist child agents for delegation |

## Agent files

An agent is a directory under `src/mastra/agents/<name>/`. Each file in it maps to one part of the agent.

- [Agent config (`config.ts`)](/docs/filesystem/config): choose the model and set runtime options
- [Instructions](/docs/filesystem/instructions): author the always-on system prompt
- [Tools](/docs/filesystem/tools): add typed functions the model can call
- [Skills](/docs/filesystem/skills): add procedures the model loads when useful
- [Memory](/docs/filesystem/memory): give the agent persistent memory
- [Workspace](/docs/filesystem/workspace): give the agent a filesystem and sandbox
- [Processors](/docs/filesystem/processors): sanitize or reshape input and output
- [Subagents](/docs/filesystem/subagents): declare specialist agents to delegate to

## Project files

Other conventions configure the project as a whole.

- [Workflows](/docs/filesystem/workflows): register workflows from `workflows/*.ts`
- [Storage](/docs/filesystem/storage): set the default store from `storage.ts`
- [Observability](/docs/filesystem/observability): configure traces, logs, and metrics from `observability.ts`
- [Server](/docs/filesystem/server): configure the HTTP server from `server.ts`
- [Studio](/docs/filesystem/studio): configure Mastra Studio from `studio.ts`

## When to use the filesystem

Use file convention when you want one place per concern, with an agent's configuration, instructions, tools, and skills grouped in a single directory. It works well when capabilities grow over time because each file or folder adds one focused behavior.

Keep using code, such as [`new Agent()`](/reference/agents/agent) and [`new Mastra()`](/reference/core/mastra-class), when you need dynamic configuration, programmatic registration, or a shared instance across modules. Both approaches coexist: code-registered primitives are always present, and the bundler adds file-based ones. On a name collision, the code-registered version wins and a warning is logged.

## How discovery works

File-based primitives are discovered by the Mastra **bundler**, which runs under `mastra dev` and `mastra build`. Markdown files are read during bundling, and `workspace/` seed files are copied into the bundled workspace so the agent starts with those files on disk.

Start your app through the Mastra CLI so discovery runs:

```bash npm2yarn
npx mastra dev
```

If you import your `mastra` instance directly, `agents/<name>/` directories and the other conventions aren't discovered. When you consume Mastra as a library, register those primitives in code instead.

Discovery is source-based and conservative. It skips symlinks, ignores test files, treats workflow and singleton files as file-routed only when they have a default export, and ignores directories that aren't agent directories.

After discovery, a file-based agent runs as a normal [`Agent`](/reference/agents/agent). The filesystem layout is an authoring convention; the assembled agent uses the same runtime as a code-defined agent.

## Get started

Create an agent directory with a `config.ts` and an `instructions.md`, then run `mastra dev`. See [agent config](/docs/filesystem/config) for the shortest working example.
