# Skill registries

:::note
The Agent Builder is a Mastra Enterprise feature. It runs without a license in local development, but production deployments require a valid `MASTRA_EE_LICENSE`. The Mastra server refuses to start with a Builder config and no valid license.
:::

Registries let the Agent Builder browse and install community skills directly from the UI. All registries are opt-in. When no registry is enabled, the Builder hides registry browse UI entirely.

## Quickstart

Enable the [skills.sh](https://skills.sh) registry:

```typescript title="src/mastra/index.ts"
import { MastraEditor } from '@mastra/editor'

new MastraEditor({
  builder: {
    enabled: true,
    registries: {
      skillsSh: { enabled: true },
    },
  },
})
```

The Builder library tab now exposes a **Browse** view backed by skills.sh. End users can preview a skill, then install it into a stored agent.

:::note

See the [AgentBuilderOptions reference](/reference/editor/agent-builder/agent-builder-options) for the full `registries` schema.

:::
