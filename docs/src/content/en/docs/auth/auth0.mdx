# MastraAuthAuth0

The `MastraAuthAuth0` class provides a lightweight authentication mechanism for Mastra using JSON Web Tokens (JWTs) issued by Auth0. It verifies incoming requests using Auth0â€™s JWKS endpoint and integrates with the Mastra server using the `experimental_auth` option.


## Installation

```bash copy
npm install @mastra/auth-auth0
```

## Usage example

```typescript {2,7-10} filename="src/mastra/index.ts" showLineNumbers copy
import { Mastra } from "@mastra/core/mastra";
import { MastraAuthAuth0 } from '@mastra/auth-auth0';

export const mastra = new Mastra({
  // ..
  server: {
    experimental_auth: new MastraAuthAuth0({
       domain: '<tenant.auth0.com>',
       audience: '<api-identifier>',
    })
  },
});
```

## Configuring `MastraClient`

With `experimental_auth` enabled, all requests made to Mastra using the [Mastra Client SDK](/docs/client-js/overview) will require a secret be sent in the authorization header:

```typescript filename="lib/mastra/mastra-client.ts" showLineNumbers copy
import { MastraClient } from "@mastra/client-js";

export const mastraClient = new MastraClient({
  baseUrl: "https://<mastra-api-url>",
  headers: {
    Authorization: `Bearer <auth0-token>`
  }
});
```

> See [Mastra Client SDK](/docs/client-js/overview) for more configuration options.
