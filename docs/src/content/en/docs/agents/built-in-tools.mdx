# Built-in tools

Mastra ships a set of agent-agnostic built-in tools that add interactive and organizational capabilities to any agent. These tools are defined in `@mastra/core/tools` and work with the standard Agent class — they don't require an [AgentController](/docs/agent-controller/overview).

| Tool | Purpose |
| - | - |
| [`ask_user`](#ask-the-user-a-question) | Ask the user a question and wait for their answer |
| [`submit_plan`](#submit-a-plan-for-review) | Submit a plan file for user approval |
| [`task_write`](#task-tracking) | Create or replace a structured task list |
| [`task_update`](#task-tracking) | Update one tracked task by ID |
| [`task_complete`](#task-tracking) | Mark one tracked task completed |
| [`task_check`](#task-tracking) | Check task list completion status |

## When to use built-in tools

- **Human-in-the-loop decisions:** Pause the agent to collect user input before continuing.
- **Plan review:** Present a plan for approval before execution begins.
- **Progress tracking:** Keep a visible, structured task list that survives long-running sessions and [observational-memory](/docs/memory/observational-memory) truncation.

## Ask the user a question

Import `askUserTool` and add it to the agent's toolset. The tool suspends the run, emits a `tool-call-suspended` event with the question, and resumes when you call `resumeStream()` with the user's answer.

```typescript title="src/mastra/agents/index.ts"
import { Agent } from '@mastra/core/agent'
import { askUserTool } from '@mastra/core/tools'

const agent = new Agent({
  id: 'assistant',
  name: 'Assistant',
  instructions: 'Ask the user for clarification when the request is ambiguous.',
  model,
  tools: { askUserTool },
})
```

### Handling the suspension

Stream the agent and watch for `tool-call-suspended` chunks. The `suspendPayload` contains the question and optional structured choices.

```typescript title="src/run.ts"
const stream = await agent.stream('Summarize my project')

for await (const chunk of stream.fullStream) {
  if (chunk.type === 'tool-call-suspended') {
    const { question, options } = chunk.payload.suspendPayload
    console.log(question)
    // Show options to the user if provided, then resume:
    const answer = await getUserAnswer() // your UI logic
    const resumed = await agent.resumeStream(answer, { runId: stream.runId })
    for await (const c of resumed.textStream) process.stdout.write(c)
  }
}
```

### Prompt shapes

`askUserTool` supports three prompt shapes:

- **Free-text:** Omit `options` to ask an open-ended question.
- **Single-select:** Provide `options` (an array of `{ label, description? }`) for a single-choice prompt.
- **Multi-select:** Provide `options` and set `selectionMode: 'multi_select'` so the user can pick more than one option. Resume with a string array of selected labels.

### Automatic resumption

Pair `askUserTool` with `autoResumeSuspendedTools` so the agent resumes automatically from the user's next chat message instead of requiring manual `resumeStream()` calls:

```typescript title="src/mastra/agents/index.ts"
const agent = new Agent({
  id: 'assistant',
  name: 'Assistant',
  instructions: 'Ask clarifying questions when needed.',
  model,
  tools: { askUserTool },
  memory: new Memory(),
  defaultOptions: {
    autoResumeSuspendedTools: true,
  },
})
```

See [Automatic tool resumption](/docs/agents/agent-approval#automatic-tool-resumption) for details.

:::note
Visit [`askUserTool` reference](/reference/tools/ask-user-tool) for the full input, suspend, and resume schemas.
:::

## Submit a plan for review

Import `submitPlanTool` to let the agent write a plan to a file and submit it for user review. The tool suspends the run until the user approves or rejects.

```typescript title="src/mastra/agents/index.ts"
import { Agent } from '@mastra/core/agent'
import { submitPlanTool } from '@mastra/core/tools'

const agent = new Agent({
  id: 'planner',
  name: 'Planner',
  instructions: 'Write a plan before starting any work. Submit it for approval.',
  model,
  tools: { submitPlanTool },
})
```

### Handling plan approval

The `suspendPayload` carries the `path` to the plan file. Read the file, present it to the user, and resume with an `action`:

```typescript title="src/run.ts"
const stream = await agent.stream('Refactor the auth module')

for await (const chunk of stream.fullStream) {
  if (
    chunk.type === 'tool-call-suspended' &&
    chunk.payload.toolName === 'submit_plan'
  ) {
    const { path } = chunk.payload.suspendPayload
    // Read and display the plan file, then resume:
    const resumed = await agent.resumeStream(
      { action: 'approved' },
      { runId: stream.runId },
    )
    for await (const c of resumed.textStream) process.stdout.write(c)
  }
}
```

To reject and request revisions:

```typescript
await agent.resumeStream(
  { action: 'rejected', feedback: 'Add error handling steps' },
  { runId: stream.runId },
)
```

:::note
Visit [`submitPlanTool` reference](/reference/tools/submit-plan-tool) for the full input, suspend, and resume schemas.
:::

## Task tracking

The four task tools manage a structured, durable task list for an agent run. They require [Memory](/docs/memory/overview) so the list is persisted in a thread-scoped store.

The recommended way to add task tracking is through [`TaskSignalProvider`](/reference/signals/task-signal-provider), which bundles all four tools and the [`TaskStateProcessor`](/reference/signals/task-signal-provider#getinputprocessors) in a single registration:

```typescript title="src/mastra/agents/index.ts"
import { Agent } from '@mastra/core/agent'
import { Memory } from '@mastra/memory'
import { TaskSignalProvider } from '@mastra/core/signals'

const agent = new Agent({
  id: 'coder',
  name: 'Coder',
  instructions: 'Track your progress with the task tools.',
  model,
  memory: new Memory(),
  signals: [new TaskSignalProvider()],
})
```

The Agent merges the four task tools into its toolset and registers the state-signal processor automatically.

### Manual wiring

Import the tools directly if you need to customize the toolset or skip the signal processor:

```typescript title="src/mastra/agents/index.ts"
import { Agent } from '@mastra/core/agent'
import { Memory } from '@mastra/memory'
import {
  taskWriteTool,
  taskUpdateTool,
  taskCompleteTool,
  taskCheckTool,
} from '@mastra/core/tools'

const agent = new Agent({
  id: 'coder',
  name: 'Coder',
  instructions: 'Track your progress with the task tools.',
  model,
  memory: new Memory(),
  tools: { taskWriteTool, taskUpdateTool, taskCompleteTool, taskCheckTool },
})
```

### How task tools work

- `task_write` replaces the entire task list. Each task has an `id`, `content`, `status` (`pending`, `in_progress`, `completed`), and `activeForm` (present-continuous description).
- `task_update` changes one task by its stable ID.
- `task_complete` marks one task completed by ID.
- `task_check` returns a summary with counts and incomplete task IDs.

Only one task can be `in_progress` at a time. The list is stored in the thread-scoped `threadState` storage domain and projected onto the agent's [state-signal](/docs/long-running-agents/signals) lane, so it survives observational-memory truncation.

:::note
Visit the [Task tools reference](/reference/tools/task-tools) for the full schemas.
:::

## Using built-in tools with AgentController

The [AgentController](/docs/agent-controller/overview) automatically includes `ask_user`, `submit_plan`, and the four task tools in every mode. You don't need to add them manually. See [Tool approvals](/docs/agent-controller/tool-approvals#built-in-tools) for AgentController-specific behavior like mode transitions on plan approval.
