# Channels

Channels connect your agents to messaging platforms like Slack, Discord, and Telegram. When a user sends a message on a platform, the agent receives it, processes it through the normal agent pipeline, and streams the response back to the conversation.

## When to use channels

Use channels when you want your agent to:

- Respond to messages in Slack workspaces, Discord servers, or Telegram chats
- Handle both direct messages and mentions in group conversations
- Use interactive buttons for tool approval flows
- Maintain conversation history across platform threads

## Quickstart

Configure channels directly on your agent using adapters from the [Chat SDK](https://chat-sdk.dev):

```typescript title="src/mastra/agents/support-agent.ts"
import { Agent } from '@mastra/core/agent'
import { createSlackAdapter } from '@chat-adapter/slack'

export const supportAgent = new Agent({
  id: 'support-agent',
  name: 'Support Agent',
  instructions: 'You are a helpful support assistant.',
  model: 'openai/gpt-5.4',
  channels: {
    adapters: {
      slack: createSlackAdapter({
        botToken: process.env.SLACK_BOT_TOKEN,
        signingSecret: process.env.SLACK_SIGNING_SECRET,
      }),
    },
  },
})
```

Register the agent in your Mastra instance with storage so channel state persists across restarts:

```typescript title="src/mastra/index.ts"
import { Mastra } from '@mastra/core'
import { LibSQLStore } from '@mastra/libsql'
import { supportAgent } from './agents/support-agent'

export const mastra = new Mastra({
  agents: { supportAgent },
  storage: new LibSQLStore({
    url: process.env.DATABASE_URL,
  }),
})
```

Mastra automatically generates webhook routes for each channel at `/api/agents/{agentId}/channels/{platform}/webhook`. Point your platform's webhook configuration to this URL.

## Multiple platforms

An agent can connect to multiple platforms simultaneously:

```typescript title="src/mastra/agents/multi-platform-agent.ts"
import { Agent } from '@mastra/core/agent'
import { createSlackAdapter } from '@chat-adapter/slack'
import { createDiscordAdapter } from '@chat-adapter/discord'
import { createTelegramAdapter } from '@chat-adapter/telegram'

export const multiPlatformAgent = new Agent({
  id: 'multi-platform-agent',
  name: 'Multi-Platform Agent',
  instructions: 'You are a helpful assistant available on multiple platforms.',
  model: 'openai/gpt-5.4',
  channels: {
    adapters: {
      slack: createSlackAdapter({
        botToken: process.env.SLACK_BOT_TOKEN,
        signingSecret: process.env.SLACK_SIGNING_SECRET,
      }),
      discord: {
        adapter: createDiscordAdapter({
          botToken: process.env.DISCORD_BOT_TOKEN,
          publicKey: process.env.DISCORD_PUBLIC_KEY,
        }),
        gateway: true,
      },
      telegram: createTelegramAdapter({
        botToken: process.env.TELEGRAM_BOT_TOKEN,
      }),
    },
  },
})
```

Wrap an adapter in an object to configure per-adapter options like `gateway` (WebSocket mode) and `cards` (rich card rendering). Pass a plain adapter to use defaults.

:::note

See [Channels reference](/reference/agents/channels#channeladapterconfig) for all per-adapter options.

:::

## Thread context

When a user mentions the agent mid-conversation in a channel thread, the agent may not have prior context. By default, Mastra fetches the last 10 messages from the platform on the first mention.

1. On the **first mention** in a thread, the agent fetches recent messages from the platform.
1. These messages are included in the system prompt as context.
1. After responding, the agent subscribes to the thread and has full history via Mastra's memory.
1. Subsequent messages in that thread do **not** re-fetch from the platform.

Set `threadContext: { maxMessages: 0 }` to disable this behavior. This only applies to non-DM threads.

## Multimodal attachments

When a user uploads a file in a platform message, the `inlineMedia` option controls which types are sent as file parts to the model. Everything else is described as a text summary so the agent knows about the file without crashing models that reject unsupported types.

```typescript
const agent = new Agent({
  // ...
  channels: {
    adapters: {
      /* ... */
    },
    // Default: ['image/*'] — only images are sent inline
    // For Gemini (supports video and audio natively):
    inlineMedia: ['image/*', 'video/*', 'audio/*'],
  },
})
```

You can also promote URLs in message text to file parts using `inlineLinks`. This is useful for models like Gemini that can process YouTube URLs as video:

```typescript
const agent = new Agent({
  // ...
  channels: {
    adapters: {
      /* ... */
    },
    inlineMedia: ['image/*', 'video/*'],
    inlineLinks: [
      { match: 'youtube.com', mimeType: 'video/*' },
      { match: 'youtu.be', mimeType: 'video/*' },
    ],
  },
})
```

:::note

See [Channels reference](/reference/agents/channels#inlinemedia) for the full `inlineMedia` and `inlineLinks` configuration.

:::

## Custom handlers

Override or disable default message handlers:

```typescript
const agent = new Agent({
  // ...
  channels: {
    adapters: {
      /* ... */
    },
    handlers: {
      onDirectMessage: false, // Disable DM handling
      onMention: async (thread, message, defaultHandler) => {
        console.log('Received mention:', message.text)
        await defaultHandler(thread, message)
      },
    },
  },
})
```

Each handler can be omitted (use default), set to `false` (disable), or a function that wraps or replaces the default behavior.

:::note

See [Channels reference](/reference/agents/channels#channelhandlers) for all handler options.

:::

## Tool approval

Tools with `requireApproval: true` render as interactive cards with Approve and Deny buttons:

```typescript
import { createTool } from '@mastra/core/tools'

const deleteFile = createTool({
  id: 'delete-file',
  description: 'Delete a file from the system',
  inputSchema: z.object({
    path: z.string().describe('Path to the file to delete'),
  }),
  requireApproval: true,
  execute: async ({ context }) => {
    await fs.unlink(context.path)
    return { deleted: context.path }
  },
})
```

When the agent calls this tool, users see a card with the tool name, arguments, and Approve/Deny buttons. The tool only executes after approval.

Set `cards: false` on an adapter to render tool calls as plain text instead of interactive cards. When cards are disabled and a tool requires approval, the agent uses `autoResumeSuspendedTools` to let the LLM decide based on the conversation context.

## Multi-user awareness

In group conversations, the agent automatically prefixes each message with the sender's name and platform mention ID. This helps the agent distinguish between speakers:

```
[Alice (@U123ABC)]: Can you help me with this?
[Bob (@U456DEF)]: I have a question too.
```

## Platform setup

Each platform requires specific configuration. See the Chat SDK documentation for detailed setup instructions:

- [Slack adapter](https://chat-sdk.dev/adapters/slack)
- [Discord adapter](https://chat-sdk.dev/adapters/discord)
- [Telegram adapter](https://chat-sdk.dev/adapters/telegram)

### Slack

1. Create a Slack app at [api.slack.com](https://api.slack.com/apps)
1. Enable Event Subscriptions and subscribe to `message.channels`, `message.groups`, `message.im`, and `app_mention`
1. Add OAuth scopes: `chat:write`, `channels:history`, `groups:history`, `im:history`, `users:read`
1. Install the app to your workspace
1. Set your webhook URL to `/api/agents/{agentId}/channels/slack/webhook`

### Discord

1. Create a Discord application at [discord.com/developers](https://discord.com/developers/applications)
1. Enable the bot and get the bot token
1. Enable "Message Content Intent" in the Bot settings
1. For webhooks, set the Interactions Endpoint URL to `/api/agents/{agentId}/channels/discord/webhook`
1. For Gateway mode (recommended), wrap the adapter: `{ adapter: createDiscordAdapter(), gateway: true }`

### Telegram

1. Create a bot with [@BotFather](https://t.me/botfather)
1. Get the bot token
1. Set your webhook URL using the Telegram Bot API:
   ```
   https://api.telegram.org/bot<token>/setWebhook?url=<your-url>/api/agents/{agentId}/channels/telegram/webhook
   ```
