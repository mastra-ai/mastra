# Channels

Channels connect your agents to messaging platforms like Slack, Discord, and Telegram. When a user sends a message on a platform, the agent receives it, processes it through the normal agent pipeline, and streams the response back to the conversation.

## When to use channels

Use channels when you want your agent to:

- Respond to messages in Slack workspaces, Discord servers, or Telegram chats
- Handle both direct messages and mentions in group conversations
- Use interactive buttons for tool approval flows
- Maintain conversation history across platform threads

## Quickstart

Configure channels directly on your agent using adapters from the [Chat SDK](https://github.com/vercel/chat):

```typescript title="src/mastra/agents/support-agent.ts"
import { Agent } from '@mastra/core/agent'
import { createSlackAdapter } from '@chat-adapter/slack'

export const supportAgent = new Agent({
  id: 'support-agent',
  name: 'Support Agent',
  instructions: 'You are a helpful support assistant.',
  model: 'openai/gpt-5.4',
  channels: {
    adapters: {
      slack: createSlackAdapter({
        botToken: process.env.SLACK_BOT_TOKEN,
        signingSecret: process.env.SLACK_SIGNING_SECRET,
      }),
    },
  },
})
```

Register the agent in your Mastra instance:

```typescript title="src/mastra/index.ts"
import { Mastra } from '@mastra/core'
import { supportAgent } from './agents/support-agent'

export const mastra = new Mastra({
  agents: { supportAgent },
})
```

Mastra automatically generates webhook routes for each channel at `/api/agents/{agentId}/channels/{platform}/webhook`. Point your platform's webhook configuration to this URL.

## Multiple platforms

An agent can connect to multiple platforms simultaneously:

```typescript title="src/mastra/agents/multi-platform-agent.ts"
import { Agent } from '@mastra/core/agent'
import { createSlackAdapter } from '@chat-adapter/slack'
import { createDiscordAdapter } from '@chat-adapter/discord'
import { createTelegramAdapter } from '@chat-adapter/telegram'

export const multiPlatformAgent = new Agent({
  id: 'multi-platform-agent',
  name: 'Multi-Platform Agent',
  instructions: 'You are a helpful assistant available on multiple platforms.',
  model: 'openai/gpt-5.4',
  channels: {
    adapters: {
      slack: createSlackAdapter({
        botToken: process.env.SLACK_BOT_TOKEN,
        signingSecret: process.env.SLACK_SIGNING_SECRET,
      }),
      discord: {
        adapter: createDiscordAdapter({
          botToken: process.env.DISCORD_BOT_TOKEN,
          publicKey: process.env.DISCORD_PUBLIC_KEY,
        }),
        gateway: true, // Enable Discord Gateway for DMs and mentions
      },
      telegram: createTelegramAdapter({
        botToken: process.env.TELEGRAM_BOT_TOKEN,
      }),
    },
  },
})
```

## Per-adapter options

Configure per-adapter behavior by wrapping the adapter in an object:

```typescript
const agent = new Agent({
  // ...
  channels: {
    adapters: {
      discord: {
        adapter: createDiscordAdapter({
          /* ... */
        }),
        gateway: true, // Enable Discord Gateway WebSocket
        cards: false, // Use plain text instead of rich cards
      },
      slack: createSlackAdapter({
        /* ... */
      }), // Plain adapter uses defaults
    },
  },
})
```

| Option | Type | Default | Description |
| - | - | - | - |
| `gateway` | `boolean` | `true` | Enable persistent WebSocket connections (Discord Gateway, Slack Socket Mode) |
| `cards` | `boolean` | `true` | Render tool calls as interactive cards with buttons |

### Disabling cards

Set `cards: false` on an adapter to render tool calls as plain text instead of rich cards. This is useful for platforms with card rendering issues:

```typescript
const agent = new Agent({
  // ...
  channels: {
    adapters: {
      discord: {
        adapter: createDiscordAdapter({
          /* ... */
        }),
        cards: false,
      },
    },
  },
})
```

When cards are disabled and a tool requires approval, the agent uses `autoResumeSuspendedTools` to let the LLM decide whether to proceed based on the conversation context.

## Channel config options

The top-level `channels` object supports these options:

| Option | Type | Default | Description |
| - | - | - | - |
| `adapters` | `Record<string, Adapter>` | required | Platform adapters keyed by name |
| `handlers` | `ChannelHandlers` | — | Override default message handlers |
| `inlineMedia` | `string[] \| function` | `['image/*']` | Mime type globs for attachments sent as file parts to the model |
| `inlineLinks` | `InlineLinkEntry[]` | — | Promote URLs in message text to file parts |
| `tools` | `boolean` | `true` | Include channel tools (e.g. `add_reaction`). Set to `false` for models without function calling |
| `state` | `StateAdapter` | in-memory | State adapter for subscriptions and deduplication |
| `userName` | `string` | agent's `name` | Bot display name shown in messages |
| `threadContext` | `{ maxMessages?: number }` | `{ maxMessages: 10 }` | Fetch prior messages on first mention in a thread |
| `chatOptions` | `ChatConfig` | — | Additional Chat SDK options (see below) |

```typescript
const agent = new Agent({
  name: 'Support Bot',
  channels: {
    adapters: {
      /* ... */
    },
    userName: 'Support Bot', // Defaults to agent's name if omitted
    state: createRedisState(), // Use Redis for persistent state
    chatOptions: {
      dedupeTtlMs: 600000, // 10 minute deduplication window
    },
  },
})
```

The `chatOptions` object accepts any [Chat SDK configuration](https://github.com/vercel/chat) not already exposed by Mastra, such as `dedupeTtlMs`, `fallbackStreamingPlaceholderText`, `lockScope`, and `messageHistory`.

## Thread context

When a user mentions the agent mid-conversation in a channel thread, the agent may not have prior context. By default, Mastra fetches the last 10 messages from the platform to provide context on the first mention.

```typescript
const agent = new Agent({
  channels: {
    adapters: {
      slack: createSlackAdapter({
        /* ... */
      }),
    },
    threadContext: { maxMessages: 20 }, // Fetch more context
  },
})
```

Thread context follows this flow:

1. On the **first mention** in a thread, the agent fetches recent messages from the platform.
1. These messages are included in the system prompt as context.
1. After responding, the agent subscribes to the thread and has full history via Mastra's memory.
1. Subsequent messages in that thread do **not** re-fetch from the platform.

Set `maxMessages: 0` to disable this behavior. This only applies to non-DM threads. In DMs, the agent already has full conversation history.

## Multimodal attachments

When a user uploads a file (image, video, PDF, etc.) in a platform message, the agent can receive it as a file part for the model to process. The `inlineMedia` option controls which attachment types are sent inline to the model. Everything else is described as a text summary so the agent knows about the file without crashing models that reject unsupported types.

```typescript
const agent = new Agent({
  // ...
  channels: {
    adapters: {
      /* ... */
    },
    // Default: only images are sent inline
    inlineMedia: ['image/*'],

    // Gemini supports video and audio natively
    inlineMedia: ['image/*', 'video/*', 'audio/*'],
  },
})
```

For platforms with private CDNs (e.g. Slack), the agent automatically fetches attachments with authenticated credentials from the Chat SDK. For platforms with public CDNs (e.g. Discord), the agent passes the URL directly to the model.

## Inline links

Use `inlineLinks` to promote URLs found in message text to file parts so the model can process linked content (images, videos, etc.) instead of seeing raw URL text.

Each entry matches a domain. Use a string for domains where a HEAD request determines the Content-Type, or an object to force a specific mime type. Forcing the mime type is useful for sites like YouTube where HEAD returns `text/html` but the model treats the URL as video.

```typescript
const agent = new Agent({
  // ...
  channels: {
    adapters: {
      /* ... */
    },
    inlineMedia: ['image/*', 'video/*'],
    inlineLinks: [
      // Force YouTube URLs as video (skip HEAD)
      { match: 'youtube.com', mimeType: 'video/*' },
      { match: 'youtu.be', mimeType: 'video/*' },
      // HEAD-check all other URLs, inline if mime matches inlineMedia
      '*',
    ],
  },
})
```

For string entries (including `'*'`), the resolved Content-Type is checked against `inlineMedia`. Only matching types become file parts. For object entries with a forced `mimeType`, the file part is always added regardless of the `inlineMedia` setting.

## Custom handlers

Override or disable default message handlers using the `handlers` option:

```typescript
const agent = new Agent({
  // ...
  channels: {
    adapters: {
      /* ... */
    },
    handlers: {
      // Disable direct message handling
      onDirectMessage: false,

      // Wrap the default mention handler with custom logic
      onMention: (thread, message, defaultHandler) => {
        console.log('Received mention:', message.text)
        // Call the default handler to process normally
        defaultHandler()
      },
    },
  },
})
```

| Handler | Description |
| - | - |
| `onDirectMessage` | Called when the bot receives a direct message |
| `onMention` | Called when the bot is mentioned in a channel |
| `onSubscribedMessage` | Called for messages in subscribed threads |

Each handler can be:

- Omitted: Uses the default Mastra handler
- `false`: Disables the handler entirely
- A function `(thread, message, defaultHandler) => void`: Wraps or replaces the default

## Tool approval

Tools with `requireApproval: true` render as interactive cards with Approve and Deny buttons:

```typescript
import { createTool } from '@mastra/core/tools'

const deleteFile = createTool({
  id: 'delete-file',
  description: 'Delete a file from the system',
  inputSchema: z.object({
    path: z.string().describe('Path to the file to delete'),
  }),
  requireApproval: true,
  execute: async ({ context }) => {
    await fs.unlink(context.path)
    return { deleted: context.path }
  },
})
```

When the agent calls this tool, users see a card with the tool name, arguments, and Approve/Deny buttons. The tool only executes after approval.

## Thread subscriptions

By default, agents respond to direct messages and mentions. To have an agent follow an entire thread (responding to all messages, not just mentions), use thread subscriptions.

The agent can subscribe to threads programmatically, and subscriptions persist across restarts when storage is configured.

## Multi-user awareness

In group conversations, the agent automatically prefixes each message with the sender's name and platform mention ID. This helps the agent distinguish between different speakers and address them appropriately.

For example, in a Slack thread with multiple users, the agent sees messages like:

```
[Alice (@U123ABC)]: Can you help me with this?
[Bob (@U456DEF)]: I have a question too.
```

## Platform setup

Each platform requires specific configuration. See the Chat SDK documentation for detailed setup instructions:

- [Slack adapter](https://github.com/vercel/chat/tree/main/packages/adapter-slack)
- [Discord adapter](https://github.com/vercel/chat/tree/main/packages/adapter-discord)
- [Telegram adapter](https://github.com/vercel/chat/tree/main/packages/adapter-telegram)

### Slack

1. Create a Slack app at [api.slack.com](https://api.slack.com/apps)
1. Enable Event Subscriptions and subscribe to `message.channels`, `message.groups`, `message.im`, and `app_mention`
1. Add OAuth scopes: `chat:write`, `channels:history`, `groups:history`, `im:history`, `users:read`
1. Install the app to your workspace
1. Set your webhook URL to `/api/agents/{agentId}/channels/slack/webhook`

### Discord

1. Create a Discord application at [discord.com/developers](https://discord.com/developers/applications)
1. Enable the bot and get the bot token
1. Enable "Message Content Intent" in the Bot settings
1. For webhooks, set the Interactions Endpoint URL to `/api/agents/{agentId}/channels/discord/webhook`
1. For Gateway mode (recommended), wrap the adapter: `{ adapter: createDiscordAdapter(), gateway: true }`

### Telegram

1. Create a bot with [@BotFather](https://t.me/botfather)
1. Get the bot token
1. Set your webhook URL using the Telegram Bot API:
   ```
   https://api.telegram.org/bot<token>/setWebhook?url=<your-url>/api/agents/{agentId}/channels/telegram/webhook
   ```

## Storage

When Mastra storage is configured, thread subscriptions and mappings persist across restarts. Without storage, the agent uses in-memory state and subscriptions are lost on restart.

```typescript title="src/mastra/index.ts"
import { Mastra } from '@mastra/core'
import { LibSQLStore } from '@mastra/libsql'

export const mastra = new Mastra({
  agents: { supportAgent },
  storage: new LibSQLStore({
    url: process.env.DATABASE_URL,
  }),
})
```
