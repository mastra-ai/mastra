# Factory rules

Factory rules are trusted deployment code that governs Work and Review board transitions, completed tool results, and normalized GitHub events. Use them to change Factory policy without moving authority into the browser or repository code.

## Quickstart

Create a versioned rules tree and pass it to `MastraFactory`. `defaultFactoryRules()` starts with the conservative built-in policy and applies your overrides.

```typescript title="src/mastra/index.ts"
import { MastraFactory } from '../web/factory-entry'
import { defaultFactoryRules } from '../web/factory/rules'

const rules = defaultFactoryRules({
  version: '2026-07-18.1',
  overrides: {
    tools: {
      submit_plan: {
        // Disable the built-in Planning to Building transition.
        onResult: () => undefined,
      },
    },
  },
})

export const factory = new MastraFactory({ rules })
```

Change `version` whenever deployed policy changes. Factory stores the version with each evaluation for audit and replay diagnostics. The version isn't an event deduplication key.

## Rule leaves

The rules tree has four namespaces:

- `work.<stage>.<source>.onEnter` and `onExit`: Govern Work board transitions.
- `review.<stage>.<source>.onEnter` and `onExit`: Govern Review board transitions independently from Work.
- `tools.<toolName>.onResult`: Evaluate a completed tool result for the exact bound Factory agent.
- `github.<event>.onEvent`: Evaluate a normalized, signed GitHub event.

A handler returns one `FactoryRuleDecision` or `undefined`. A decision can have one of these effects:

- Reject the ingress
- Request another governed transition
- Upsert a linked card
- Invoke a skill
- Send a message to a bound role
- Create a notification

External effects are persisted and dispatched after the authoritative transaction commits.

Handlers receive bounded context. They don't receive storage handles, credentials, worktree paths, or raw webhook payloads. GitHub actors are trusted only when repository permission resolves to `write` or `admin`. Permission errors fail closed.

## Replace an exact leaf

Overrides replace only the specified `onEnter`, `onExit`, `onResult`, or `onEvent` leaf. They don't compose with the built-in handler at that leaf, and they don't replace sibling leaves.

The following override rejects a Review card moving to Done until an injected checks service reports success:

```typescript title="src/mastra/index.ts"
import { deploymentChecks } from './deployment-checks'

const rules = defaultFactoryRules({
  version: '2026-07-18.2',
  overrides: {
    review: {
      done: {
        pullRequest: {
          onEnter: async context => {
            const complete = await deploymentChecks.areComplete(context.item.url)

            return complete
              ? undefined
              : {
                  type: 'reject',
                  code: 'forbidden',
                  reason: 'Required pull request checks are incomplete.',
                }
          },
        },
      },
    },
  },
})
```

Rules have a five-second evaluation budget. Keep network reads bounded and return a short rejection reason that's safe to persist and display.

This example disables built-in automatic PR review for selected repositories. A GitHub PR source key includes the numeric repository ID.

```typescript title="src/mastra/index.ts"
const repositoriesWithoutAutoReview = new Set([123456789])

const rules = defaultFactoryRules({
  version: '2026-07-18.3',
  overrides: {
    review: {
      intake: {
        pullRequest: {
          onEnter: context => {
            const repositoryId = Number(context.item.sourceKey?.split(':')[1])

            if (repositoriesWithoutAutoReview.has(repositoryId)) return undefined

            return {
              type: 'invokeSkill',
              idempotencyKey: `${context.ingress.id}:understand-pr`,
              role: 'review',
              skillName: 'understand-pr',
            }
          },
        },
      },
    },
  },
})
```

Returning `undefined` allows the transition but performs no additional action. A rejection blocks the transition and preserves the canonical card state.
