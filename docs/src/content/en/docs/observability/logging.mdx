# Logging

Mastra's logging system captures function execution, input data, and output responses in a structured format.

## PinoLogger

When [initializing a new Mastra project](../getting-started/installation.mdx) using the CLI, `PinoLogger` is included by default and configured to print informational messages to the [Mastra Cloud Logs](../mastra-cloud//observability.mdx).


```typescript filename="src/mastra/index.ts" showLineNumbers copy
import { Mastra } from '@mastra/core/mastra';
import { PinoLogger } from '@mastra/loggers';

export const mastra = new Mastra({
  // ...
  logger: new PinoLogger({
    name: 'Mastra',
    level: 'info',
  }),
});
```

> See the [PinoLogger](../../reference//observability/logger.mdx) API reference for all available configuration options.

### Configuration explained

- **name** – Sets the logger name used to group and identify logs.
- **level** – Defines the minimum log level to record.

## Logging from workflow steps

To capture detailed logs during workflow execution, use the logger instance available via the `mastra` parameter inside a step’s `execute` function. Use severity-based methods such as `info` and `error` to record relevant events during runtime.

```typescript {6,9,15,18} filename="src/mastra/workflows/test-workflow.ts" showLineNumbers copy
import { createWorkflow, createStep } from "@mastra/core/workflows";
import { z } from "zod";

const step1 = createStep({
  //...
  execute: async ({ inputData, mastra }) => {
    const { input } = inputData;

    const logger = mastra.getLogger();

    const agent = mastra.getAgent("testAgent");
    const response = await agent.generate(input);

    if (!agent) {
      logger.error("This is an error log");
    }

    logger.info("This is an info log");

    return {
      output: response.text
    };
  }
});

export const testWorkflow = createWorkflow({...})
  .then(step1)
  .commit();
```
