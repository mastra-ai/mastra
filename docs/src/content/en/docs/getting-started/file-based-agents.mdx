# File-based Agents overview

:::experimental

This feature is in beta. Breaking changes may occur without a major version bump until the API is stable.

:::

With file-based routing, the directory is the agent. Each folder under `src/mastra/agents/` becomes an [`Agent`](/reference/agents/agent) instance, its directory name becomes the default `id` and `name`, and each file inside the folder adds one capability.

A file-based project starts with `config.ts` and `instructions.md`. Add more files only when the agent needs more capabilities.

## Agent files

File-based agents use a default-first model. Start with the files required to run, then add folders as the agent grows. Each file in `src/mastra/agents/<name>/` maps to one part of the agent.

| File or folder | What it adds |
| - | - |
| [`config.ts`](/reference/file-based-agents/config) | Model selection and runtime options |
| [`instructions.md`](/reference/file-based-agents/instructions) | The always-on system prompt |
| [`tools/`](/reference/file-based-agents/tools) | Typed functions the model can call |
| [`skills/`](/reference/file-based-agents/skills) | Procedures loaded on demand |
| [`memory.ts`](/reference/file-based-agents/memory) | Persistent conversation memory |
| [`workspace.ts`](/reference/file-based-agents/workspace) | A filesystem and sandbox |
| [`workspace/`](/reference/file-based-agents/workspace#seed-files) | Seed files shipped with the agent |
| [`processors/`](/reference/file-based-agents/processors) | Input/output processing pipeline |
| [`subagents/`](/reference/file-based-agents/subagents) | Specialist child agents for delegation |

## Project files

Other conventions configure the project as a whole.

- [Workflows](/reference/file-based-agents/workflows): register workflows from `workflows/*.ts`
- [Storage](/reference/file-based-agents/storage): set the default store from `storage.ts`
- [Observability](/reference/file-based-agents/observability): configure traces, logs, and metrics from `observability.ts`
- [Server](/reference/file-based-agents/server): configure the HTTP server from `server.ts`
- [Studio](/reference/file-based-agents/studio): configure Mastra Studio from `studio.ts`

## When to use file-based agents

Use file convention when you want one place per concern, with an agent's configuration, instructions, tools, and skills grouped in a single directory. It works well when capabilities grow over time because each file or folder adds one focused behavior.

Keep using code, such as [`new Agent()`](/reference/agents/agent) and [`new Mastra()`](/reference/core/mastra-class), when you need dynamic configuration, programmatic registration, or a shared instance across modules. Both approaches coexist: code-registered primitives are always present, and the bundler adds file-based ones. On a name collision, the code-registered version wins and a warning is logged.

## How discovery works

File-based primitives are discovered by the Mastra **bundler**, which runs under `mastra dev` and `mastra build`. Markdown files are read during bundling, and `workspace/` seed files are copied into the bundled workspace so the agent starts with those files on disk.

Start your app through the Mastra CLI so discovery runs:

```bash npm2yarn
npx mastra dev
```

If you import your `mastra` instance directly, `agents/<name>/` directories and the other conventions aren't discovered. When you consume Mastra as a library, register those primitives in code instead.

Discovery is source-based and conservative. It skips symlinks, ignores test files, treats workflow and singleton files as file-routed only when they have a default export, and ignores directories that aren't agent directories.

After discovery, a file-based agent runs as a normal [`Agent`](/reference/agents/agent). The file-based layout is an authoring convention; the assembled agent uses the same runtime as a code-defined agent.

## Get started

Create an agent directory with a `config.ts` and an `instructions.md`, then run `mastra dev`. The following `config.ts` plus an `instructions.md` file creates a working file-based agent:

```typescript title="src/mastra/agents/weather/config.ts"
import { agentConfig } from '@mastra/core/agent'

export default agentConfig({
  model: '__GATEWAY_OPENAI_MODEL__',
})
```

```markdown title="src/mastra/agents/weather/instructions.md"
You are a helpful weather assistant. Answer questions about current conditions and forecasts.
```

Mastra uses the `weather` directory name as the default agent `id` and `name`. The sibling `instructions.md` supplies the required instructions.
