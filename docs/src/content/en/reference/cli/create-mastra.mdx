# create-mastra

Create a standalone Mastra project. By default, `create-mastra` installs the managed agent harness and configures it for your selected model provider.

```bash npm2yarn
npx create-mastra@latest my-mastra-project --yes
```

You can also run the command without arguments to enter the interactive flow:

```bash npm2yarn
npx create-mastra@latest
```

## Creation modes

### Managed agent harness

The default mode creates an agent harness with workspace tools, memory, task tracking, web access, recurring schedules, storage, and observability. Select OpenAI, Anthropic, Google Gemini, or xAI:

```bash npm2yarn
npx create-mastra@latest my-agent --llm anthropic --yes
```

The generated provider dependency, models, API key variable, and native search tool match the selected provider. xAI projects omit native web search.

When you don't use `--yes`, the command prompts for a missing project name, provider, and optional API key. When you use `--yes`, you must provide the project name; the provider defaults to OpenAI if you don't pass `--llm`.

### Template

Use a template slug or a public GitHub URL:

```bash npm2yarn
npx create-mastra@latest my-project --template coding-agent
```

```bash npm2yarn
npx create-mastra@latest my-project --template https://github.com/mastra-ai/template-agent-harness
```

Leave the template value blank to select a template interactively:

```bash npm2yarn
npx create-mastra@latest my-project --template
```

Template authors own their dependencies, models, environment variables, and source code. `create-mastra` doesn't apply `--llm`, `--llm-api-key`, or managed agent-harness changes to an arbitrary template.

### Empty scaffold

Use `--empty` to create a provider-free project without agents, examples, model SDKs, or environment files:

```bash npm2yarn
npx create-mastra@latest my-empty-project --empty --yes
```

## Automatic setup

After installing dependencies, the command:

1. Detects supported coding assistants on `PATH` and installs Mastra skills. If none are detected, it installs universal skills.
1. Creates an initial Git commit when the current directory and generated project aren't already inside Git repositories.

Use `--no-skills` or `--no-git` to skip these steps. Skills and Git setup failures produce warnings but don't remove a successfully created project.

## Conflicts and validation

- `--empty` and `--template` can't be used together.
- `--llm` and `--llm-api-key` are only valid for the managed agent harness.
- A bare `--template` can't be used with `--yes` because it requires template selection.
- `--yes` requires a positional project name.
- The project name must be a safe, lowercase, single directory name and the target must not already exist.

Invalid input is rejected before templates are fetched or files are created.

## Telemetry

Mastra collects anonymous CLI usage information, such as the operating system, Mastra version, and Node.js version. You can review the [analytics source](https://github.com/mastra-ai/mastra/blob/main/packages/cli/src/analytics/index.ts).

Set `MASTRA_TELEMETRY_DISABLED=1` to opt out:

```bash
MASTRA_TELEMETRY_DISABLED=1 npx create-mastra@latest my-project --yes
```
