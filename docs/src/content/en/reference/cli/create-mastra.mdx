# create-mastra

The `create-mastra` command creates a new standalone Mastra project in a dedicated directory. You can use the interactive setup, a fixed default preset, custom non-interactive options, or a project template.

## Before you begin

Install Node.js `22.13.0` or later.

## Usage

### Interactive setup

Run the command without options to start the setup wizard:

```bash npm2yarn
npx create-mastra@latest
```

### Default setup

Use `--default` to create a project with the `src/` directory, OpenAI, agents, tools, workflows, scorers, and example code:

```bash npm2yarn
npx create-mastra@latest my-mastra-project --default
```

The default preset ignores `--components`, `--llm`, `--llm-api-key`, `--example`, `--no-example`, and `--dir`. Use custom non-interactive setup when you need to change those values.

### Custom non-interactive setup

To skip the setup wizard and preserve custom scaffold options, provide a project name, `--llm`, and either `--example` or `--no-example`:

```bash npm2yarn
npx create-mastra@latest my-mastra-project --components agents,tools,workflows,scorers --llm anthropic --example
```

When you include `workflows` and `--example`, also include `agents`. The generated example workflow uses the generated weather agent.

### Template setup

Use a template `slug` from the [templates list](https://mastra.ai/api/templates.json):

```bash npm2yarn
npx create-mastra@latest my-mastra-project --template deep-search
```

You can also use the full URL of a public GitHub repository:

```bash npm2yarn
npx create-mastra@latest my-mastra-project --template https://github.com/mastra-ai/template-deep-search
```

For GitHub repositories, `create-mastra` checks the `main` and `master` branches for:

- A root `package.json` that includes `@mastra/core` as a dependency, development dependency, or peer dependency.
- A `src/mastra/index.ts` file that exports a `Mastra` instance.

Template setup prompts for a model provider unless you provide `--llm`, and always prompts whether to initialize Git. Other scaffold options don't customize template contents.

To open the interactive template selector, pass `--template` without a value:

```bash npm2yarn
npx create-mastra@latest --template
```

## Telemetry

By default, Mastra sends CLI environment and usage metadata to PostHog. This includes:

- Environment metadata: Operating system and version, Node.js version, processor architecture, package manager, CLI version, and machine hostname.
- Usage metadata: Command status and duration, plus selected command arguments such as the project name, components, provider, template, and Observability selection.

The `create-mastra` telemetry payload doesn't include the value passed to `--llm-api-key`.

You can inspect the [telemetry source code](https://github.com/mastra-ai/mastra/blob/main/packages/cli/src/analytics/index.ts).

To opt out of CLI telemetry, set `MASTRA_TELEMETRY_DISABLED`:

```bash
MASTRA_TELEMETRY_DISABLED=1
```

You can set the variable for an individual command:

```bash
MASTRA_TELEMETRY_DISABLED=1 npx create-mastra@latest
```
