# create-agentbuilder

The `create-agentbuilder` command creates a new standalone Mastra project with the [Agent Builder](/docs/agent-builder/overview) pre-configured. The scaffolded project registers the Builder agent, composes LibSQL and DuckDB storage, and enables Mastra Observability exporters.

The command prompts for a project name and whether to enable [Mastra Observability](/docs/observability/overview). The model provider is fixed to OpenAI because the Builder agent runs on an OpenAI model.

:::note
The Agent Builder is part of the Mastra Enterprise Edition. It works without a license key in local development, but production deployments require a valid EE license set via the `MASTRA_EE_LICENSE` environment variable. [Contact sales](https://mastra.ai/contact) for more information.
:::

## Usage

```bash npm2yarn
npx create-agentbuilder@latest
```

You can also pass the project name and OpenAI API key directly:

```bash npm2yarn
npx create-agentbuilder@latest my-agent-builder-app -- --llm-api-key <openai-api-key>
```

After scaffolding, start the dev server:

```bash
cd my-agent-builder-app
npm run dev
```

The Agent Builder is mounted at `http://localhost:4111/agent-builder`.

## Telemetry

By default, Mastra collects anonymous information about your project like your OS, Mastra version or Node.js version. You can read the [source code](https://github.com/mastra-ai/mastra/blob/main/packages/cli/src/analytics/index.ts) to check what's collected.

You can opt out of the CLI analytics by setting an environment variable:

```bash
MASTRA_TELEMETRY_DISABLED=1 npx create-agentbuilder@latest
```
