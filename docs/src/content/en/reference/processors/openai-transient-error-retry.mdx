# OpenAITransientErrorRetry

`OpenAITransientErrorRetry` is an **error processor** that retries transient OpenAI Responses API stream errors. Use it for agents that call OpenAI Responses models, including Codex-style providers that are built on the OpenAI provider.

The processor isn't enabled by default in core. Add it to `errorProcessors` for agents that use OpenAI Responses or Codex models.

## Usage example

Add `OpenAITransientErrorRetry` to `errorProcessors` for OpenAI Responses agents:

```typescript title="src/mastra/agents/openai-agent.ts"
import { Agent } from '@mastra/core/agent'
import { OpenAITransientErrorRetry } from '@mastra/core/processors'

export const agent = new Agent({
  name: 'openai-agent',
  instructions: 'You are a helpful assistant.',
  model: 'openai/gpt-5',
  errorProcessors: [new OpenAITransientErrorRetry()],
})
```

## How it works

The processor checks the error and its cause chain for:

- Provider retry metadata: `isRetryable === true`
- Retryable HTTP status codes: `408`, `409`, `429`, and `5xx`
- OpenAI Responses stream error chunks: `type: 'error'` and `type: 'response.failed'`
- Explicit OpenAI retry guidance: Error messages that include `You can retry your request`

When the error is transient, the processor returns `{ retry: true }`. It doesn't mutate messages.
