# ModelSwapperProcessor

The `ModelSwapperProcessor` is an **input processor** that classifies the latest user request and swaps the agent model before the first language model call. Use it to route short or routine requests to lower-cost models while preserving stronger models for requests that need more reasoning.

The processor runs classification once in `processInput()` and applies the selected model in `processInputStep()`. Messages are not rewritten.

## Usage example

The following example demonstrates routing requests to different models with a small classifier model:

```typescript title="src/mastra/agents/support-agent.ts"
import { Agent } from '@mastra/core/agent'
import { ModelSwapperProcessor } from '@mastra/core/processors'

export const supportAgent = new Agent({
  name: 'Support Agent',
  instructions: 'Answer user questions and provide detailed help when needed.',
  model: '__GATEWAY_OPENAI_MODEL_BASE__',
  inputProcessors: [
    new ModelSwapperProcessor({
      model: '__GATEWAY_OPENAI_MODEL_NANO__',
      defaultModel: '__GATEWAY_OPENAI_MODEL_BASE__',
      rules: [
        {
          description: 'Short factual questions, summaries, and routine support requests',
          model: '__GATEWAY_OPENAI_MODEL_MINI__',
        },
        {
          description: 'Architecture, debugging, planning, or multi-step analysis',
          model: '__GATEWAY_OPENAI_MODEL__',
        },
      ],
    }),
  ],
})
```

## Routing rules

Each `ModelSwapRule` describes when a request should use a specific model. Rules are ordered. The classifier returns the rule number, where `1` maps to `rules[0]`, `2` maps to `rules[1]`, and `0` means no match.

## Result metadata

The processor stores the classification result in its request-scoped state. The result has the following shape:

```typescript
type ModelSwapperResult = {
  rule: number
  selectedModel?: MastraModelConfig
}
```

`selectedModel` is set when a valid rule number matches, or when `defaultModel` is configured for unmatched requests.

## Behavior notes

- Classification uses the latest message by default. Set `lastMessageOnly: false` to classify all selected input messages.
- The selected model is applied only on step `0`. Tool-continuation steps keep the current model unless another processor changes it.
- If the classifier returns `0` and no `defaultModel` is configured, the processor returns `{}` from `processInputStep()` and the agent's configured model stays unchanged.
- If the classifier fails, the processor fails open and leaves the model unchanged.
- Message content is passed through unchanged.
