# Mastra.getStorage()

The `.getStorage()` method is used to retrieve the storage instance that has been configured in the Mastra instance.

## Usage example

```typescript copy
mastra.getStorage();
```

## Parameters

This method does not accept any parameters.

## Returns

<PropertiesTable
  content={[
    {
      name: "storage",
      type: "MastraStorage | undefined",
      description:
        "The configured storage instance, or undefined if no storage has been configured.",
    },
  ]}
/>

## Accessing Storage Domains

To access a specific storage domain (memory, workflows, evals, or observability), use the `.getStore()` method instead:

```typescript copy
// Get the memory domain store
const memoryStore = await mastra.getStore('memory');

// Get the workflows domain store
const workflowsStore = await mastra.getStore('workflows');
```

The `.getStore()` method is async and returns a promise. See the [`.getStore()` reference](/reference/v1/core/getStore) for detailed documentation, or the [Storage overview](/docs/v1/server-db/storage) for more information about storage domains.
