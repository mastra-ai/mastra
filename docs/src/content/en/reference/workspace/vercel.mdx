# VercelSandbox

Executes commands in ephemeral [Vercel Sandbox](https://vercel.com/docs/vercel-sandbox) microVMs. Each sandbox is a full Firecracker VM with a persistent filesystem, real shell with sudo access, and support for background processes.

:::info

For interface details, see [WorkspaceSandbox interface](/reference/workspace/sandbox).

:::

## Installation

```bash npm2yarn
npm install @mastra/vercel
```

## Usage

Add a `VercelSandbox` to a workspace and assign it to an agent:

```typescript
import { Agent } from '@mastra/core/agent'
import { Workspace } from '@mastra/core/workspace'
import { VercelSandbox } from '@mastra/vercel'

const workspace = new Workspace({
  sandbox: new VercelSandbox({
    runtime: 'node24',
  }),
})

const agent = new Agent({
  name: 'dev-agent',
  model: 'anthropic/claude-sonnet-4-6',
  workspace,
})
```

### Custom resources and port exposure

```typescript
const workspace = new Workspace({
  sandbox: new VercelSandbox({
    runtime: 'node24',
    vcpus: 4,
    timeout: 600_000,
    ports: [3000, 8080],
    env: {
      NODE_ENV: 'production',
    },
  }),
})
```

## Methods

### Lifecycle

#### `start()`

Creates the Vercel Sandbox microVM. Called automatically by the base class lifecycle.

```typescript
await sandbox._start()
```

#### `stop()`

Stops the microVM. The sandbox cannot be reused after stopping.

```typescript
await sandbox._stop()
```

#### `destroy()`

Stops the microVM with blocking, ensuring full cleanup.

```typescript
await sandbox._destroy()
```

### Info

#### `getInfo()`

Returns current sandbox metadata including the Vercel sandbox ID, runtime, and VM status.

```typescript
const info = await sandbox.getInfo()
// { id, name, provider, status, createdAt, metadata: { sandboxId, runtime, vcpus, timeout, vmStatus } }
```

#### `getInstructions(opts?)`

Returns instructions describing the sandbox capabilities for use in agent system prompts.

```typescript
const instructions = sandbox.getInstructions()
```

## Authentication

Vercel Sandbox requires authentication. Either:

1. **OIDC (recommended)**: Link your Vercel project and pull credentials:
   ```bash
   npx vercel link
   npx vercel env pull
   ```

1. **Manual credentials**: Provide `VERCEL_TOKEN`, `VERCEL_TEAM_ID`, and `VERCEL_PROJECT_ID` as environment variables.

## Limitations

- **No cloud storage mounting**: S3/GCS/R2 FUSE mounts are not supported. Use `writeFiles`/`readFileToBuffer` on the Vercel SDK directly.
- **No stdin**: Interactive process input (`sendStdin`) is not supported by the Vercel Sandbox SDK.
- **Sandbox timeout**: MicroVMs have a maximum lifetime bounded by `timeout` (default 5 minutes).
