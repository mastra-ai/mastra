# Route Contracts

Utility types that infer path params, query params, request body, and response types from any Mastra server route. Import them from `@mastra/server/schemas`.

When a new route is added via `createRoute()`, it automatically appears in the `RouteMap` type without manual updates.

## Usage example

Infer the types of any route by looking it up in the `RouteMap` with a `"METHOD /path"` key.

```typescript
import type {
  RouteMap,
  RouteContract,
  InferPathParams,
  InferBody,
  InferResponse,
} from '@mastra/server/schemas'

// Look up a route by method + path
type GetAgentRoute = RouteContract<'GET /agents/:agentId'>

// Infer individual schemas
type AgentPathParams = InferPathParams<GetAgentRoute>
// => { agentId: string }

type GenerateBody = InferBody<RouteContract<'POST /agents/:agentId/generate'>>
// => { messages: CoreMessage[] | string, instructions?: string, ... }

type AgentResponse = InferResponse<RouteContract<'GET /agents/:agentId'>>
// => { name: string, tools: Record<string, ...>, ... }
```

Invalid route keys are rejected at compile time:

```typescript
// @ts-expect-error — this key doesn't exist
type Bad = RouteContract<'GET /this/does/not/exist'>
```

## Import

```typescript
import type {
  RouteMap,

} from '@mastra/server/schemas'
```

## Types

### `RouteMap`

A mapped type keyed by `"METHOD /path"` strings. Each value is the corresponding `ServerRoute` with full schema type information.

```text
type RouteMap = {
  'GET /agents': ServerRoute<...>
  'GET /agents/:agentId': ServerRoute<...>
  'POST /agents/:agentId/generate': ServerRoute<...>
  // ... all registered routes
}
```

### `RouteContract<K>`

Shorthand for `RouteMap[K]`. Retrieves a single route type from the map.

```typescript
type Route = RouteContract<'GET /agents/:agentId'>
```

### `InferPathParams<R>`

Extracts the path parameter types from a route's `pathParamSchema`.

```typescript
type Params = InferPathParams<RouteContract<'GET /agents/:agentId'>>
// => { agentId: string }
```

**Returns:** The inferred Zod output type from the path param schema, or `never` if the route has no path params.

### `InferQueryParams<R>`

Extracts the query parameter types from a route's `queryParamSchema`.

```typescript
type Query = InferQueryParams<RouteContract<'GET /agents'>>
// => { partial?: string }
```

**Returns:** The inferred Zod output type from the query param schema, or `never` if the route has no query params.

### `InferBody<R>`

Extracts the request body types from a route's `bodySchema`.

```typescript
type Body = InferBody<RouteContract<'POST /agents/:agentId/generate'>>
// => { messages: CoreMessage[] | string, ... }
```

**Returns:** The inferred Zod output type from the body schema, or `never` if the route has no body schema.

### `InferResponse<R>`

Extracts the response types from a route's `responseSchema`.

```typescript
type Response = InferResponse<RouteContract<'GET /agents/:agentId'>>
// => { name: string, tools: Record<string, ...>, ... }
```

**Returns:** The inferred Zod output type from the response schema, or `never` if the route has no response schema.

## Additional exports

The `@mastra/server/schemas` entry point also re-exports all Zod validation schemas used by the server routes. You can use these directly for runtime validation:

```typescript
import { agentIdPathParams, agentExecutionBodySchema } from '@mastra/server/schemas'

// Use Zod schemas for validation
const parsed = agentExecutionBodySchema.parse(requestBody)
```
