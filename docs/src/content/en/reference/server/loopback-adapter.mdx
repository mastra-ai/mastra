# LoopBack adapter

The `@mastra/loopback` package provides a server adapter for running Mastra with [LoopBack 4](https://loopback.io/doc/en/lb4/).

:::info

For general adapter concepts such as initialization flow and route registration, see [Server Adapters](/docs/server/server-adapters).

:::

## Installation

Install the LoopBack adapter and required LoopBack packages:

```bash npm2yarn
npm install @mastra/loopback@latest @loopback/core @loopback/rest
```

## Usage example

The following example demonstrates how to attach Mastra to a native LoopBack `RestApplication`.

```typescript title="src/index.ts"
import { RestApplication } from '@loopback/rest'
import { LoopbackMastraServer } from '@mastra/loopback'
import { mastra } from './mastra'

const app = new RestApplication()

const server = new LoopbackMastraServer({
  app,
  mastra,
  config: {
    prefix: '/api/mastra',
    openapiPath: '/openapi.json',
  },
})

await server.init()
await app.start()
```

## Resolve LoopBack bindings from Mastra

The adapter injects a LoopBack bridge into Mastra `RequestContext` under the `loopback` key.

```typescript
const loopback = requestContext.get('loopback')
const customerService = await loopback.resolve('services.CustomerService')
```

The bridge exposes:

- `app`
- `context`
- `request`
- `response`
- `resolve(binding)`
- `resolveSync(binding)`
- `isBound(binding)`

Use this bridge to access request-scoped LoopBack services, repositories, and other bindings from Mastra routes, tools, and agents.

## Custom routes

Mastra `apiRoutes` are registered as native LoopBack routes.

```typescript title="src/mastra/index.ts"
import { registerApiRoute } from '@mastra/core/server'

mastra.setServer({
  apiRoutes: [
    registerApiRoute('/customer/:id', {
      method: 'GET',
      handler: async c => {
        const requestContext = c.get('requestContext')
        const loopback = requestContext.get('loopback')
        const customerService = await loopback.resolve('services.CustomerService')

        return c.json(customerService.findById(c.req.param('id')))
      },
      openapi: {
        summary: 'Get customer by id',
      },
    }),
  ],
})
```

## Auth configuration

The adapter lets you compose custom authorization with Mastra route auth.

```typescript title="src/index.ts"
const server = new LoopbackMastraServer({
  app,
  mastra,
  config: {
    prefix: '/api/mastra',
    auth: {
      authorizeMode: 'after',
      authorize: async input => {
        return input.getHeader('x-tenant-id')
          ? null
          : { status: 403, error: 'Tenant header required' }
      },
      resolveContextMode: 'replace',
      resolveContext: async input => ({
        userId: input.headers['x-user-id'] as string | undefined,
      }),
    },
  },
})
```

Supported composition modes:

- `authorizeMode`: `before`, `after`, `replace`
- `resolveContextMode`: `before`, `after`, `replace`
