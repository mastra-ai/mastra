# Conversations API

The Conversations API adds a `conversations` namespace to `MastraClient` so you can create a thread-backed conversation and retrieve its stored messages.

In Mastra, a conversation ID is the raw memory thread ID. Responses API calls can return this value as `conversation_id`, and the Conversations API lets you create and inspect the same thread directly.

## Usage example

Create a conversation:

```typescript
import { MastraClient } from '@mastra/client-js'

const client = new MastraClient({
  baseUrl: 'http://localhost:4111',
})

const conversation = await client.conversations.create({
  agent_id: 'support-agent',
})

console.log(conversation.id)
```

Retrieve a conversation and its stored messages:

```typescript
const conversation = await client.conversations.retrieve('thread_123')

console.log(conversation.messages)
```

## Methods

### Lifecycle

#### `create(params)`

Creates a new conversation thread for the selected agent.

```typescript
const conversation = await client.conversations.create({
  agent_id: 'support-agent',
  title: 'Billing support',
})
```

**Returns:** `Promise<Conversation>`.

#### `retrieve(conversationId, requestContext?)`

Retrieves a conversation by its thread ID and includes the stored messages in that thread.

```typescript
const conversation = await client.conversations.retrieve('thread_123')

console.log(conversation.thread)
console.log(conversation.messages)
```

**Returns:** `Promise<Conversation>`.

## Conversation shape

A conversation object includes:

- `id`: The raw thread ID
- `object`: Always `'conversation'`
- `thread`: The stored thread record
- `messages`: The messages currently stored in that thread

Use this API when you want to create a conversation before the first Responses request, or when you want to inspect the full thread behind a stored Responses flow.
