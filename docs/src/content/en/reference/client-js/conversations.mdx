# Conversations API

The Conversations API adds a `conversations` namespace to `MastraClient` so you can create a thread-backed conversation and retrieve its stored items.

In Mastra, a conversation ID is the raw memory thread ID. Responses API calls can return this value as `conversation_id`, and the Conversations API lets you create and inspect the same thread directly.

## Usage example

Create a conversation:

```typescript
import { MastraClient } from '@mastra/client-js'

const client = new MastraClient({
  baseUrl: 'http://localhost:4111',
})

const conversation = await client.conversations.create({
  agent_id: 'support-agent',
})

console.log(conversation.id)
```

Retrieve the stored items for a conversation:

```typescript
const items = await client.conversations.retrieve('thread_123')

console.log(items.data)
```

## Methods

### Lifecycle

#### `create(params)`

Creates a new conversation thread for the selected agent.

```typescript
const conversation = await client.conversations.create({
  agent_id: 'support-agent',
  title: 'Billing support',
})
```

**Returns:** `Promise<Conversation>`.

#### `retrieve(conversationId, requestContext?)`

Retrieves the item list for a conversation by its thread ID.

```typescript
const items = await client.conversations.retrieve('thread_123')

console.log(items.data)
```

**Returns:** `Promise<ConversationItemsPage>`.

## Response shapes

`create()` returns a conversation object with:

- `id`: The raw thread ID
- `object`: Always `'conversation'`
- `thread`: The stored thread record

`retrieve()` returns an item list with:

- `object`: Always `'list'`
- `data`: Conversation items such as `message`, `function_call`, and `function_call_output`
- `first_id`: The first item ID in the page
- `last_id`: The last item ID in the page
- `has_more`: Whether more items exist beyond this page

Use this API when you want to create a conversation before the first Responses request, or when you want to inspect the stored item history behind a Responses flow.
