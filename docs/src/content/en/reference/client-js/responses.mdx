# Responses API

The Responses API adds an OpenAI-style namespace to `MastraClient` so you can call Mastra's built-in `/v1/responses` routes with `client.responses.create()`, `client.responses.retrieve()`, `client.responses.stream()`, and `client.responses.delete()`.

These routes are a Mastra agent-backed adapter layer. They expose an OpenAI-like Responses surface while still executing through registered Mastra agents, agent memory, and Mastra storage.

`model` is the target model string, such as `openai/gpt-5`, and `agent_id` selects the registered Mastra agent that executes the request. Initial requests must provide `agent_id`. Stored follow-up turns can omit it when continuing with `previous_response_id`.

Stored responses also return `conversation_id`. In Mastra, this is the raw memory `threadId`. You can pass `conversation_id` on future requests to continue the same conversation thread directly.

If you want to compare this with the official `openai` SDK path, see the [Responses API demo](https://github.com/mastra-ai/mastra/tree/main/examples/responses-api-demo).

## Usage example

Create a non-streaming response:

```typescript
import { MastraClient } from '@mastra/client-js'

const client = new MastraClient({
  baseUrl: 'http://localhost:4111',
})

const response = await client.responses.create({
  model: 'openai/gpt-5',
  agent_id: 'support-agent',
  input: 'Summarize this ticket',
  store: true,
})

console.log(response.output_text)
```

Stream response events:

```typescript
const stream = await client.responses.create({
  model: 'openai/gpt-5',
  agent_id: 'support-agent',
  input: 'Say hello',
  stream: true,
})

for await (const event of stream) {
  console.log(event.type)
}
```

Request JSON object output:

```typescript
const response = await client.responses.create({
  model: 'openai/gpt-5',
  agent_id: 'support-agent',
  input: 'Return a JSON object with a short summary and priority.',
  text: {
    format: {
      type: 'json_object',
    },
  },
})

console.log(response.output_text)
```

Request schema-constrained structured output:

```typescript
const response = await client.responses.create({
  model: 'openai/gpt-5',
  agent_id: 'support-agent',
  input: 'Return a structured support ticket summary.',
  text: {
    format: {
      type: 'json_schema',
      name: 'ticket_summary',
      strict: true,
      schema: {
        type: 'object',
        properties: {
          summary: { type: 'string' },
          priority: { type: 'string' },
        },
        required: ['summary', 'priority'],
        additionalProperties: false,
      },
    },
  },
})
```

## Methods

### Lifecycle

#### `create(params)`

Creates a response using the OpenAI-compatible route.

```typescript
const response = await client.responses.create({
  model: 'openai/gpt-5',
  agent_id: 'support-agent',
  input: 'Summarize this ticket',
})
```

**Returns:** `Promise<ResponsesResponse>` when `stream` is omitted or `false`.

Configured tool definitions are exposed on `response.tools`. If the agent calls a tool, that activity is represented in `response.output` as `function_call` and `function_call_output` items alongside the assistant `message`.

Stored responses include `conversation_id` and `response.id`. Use `conversation_id` when you want to target a known thread directly:

```typescript
const first = await client.responses.create({
  model: 'openai/gpt-5',
  agent_id: 'support-agent',
  input: 'Start a support thread',
  store: true,
})

const second = await client.responses.create({
  model: 'openai/gpt-5',
  agent_id: 'support-agent',
  conversation_id: first.conversation_id!,
  input: 'Add a follow-up to the same thread',
  store: true,
})
```

When `stream: true`, `create()` returns an async iterable of SSE event payloads:

```typescript
const stream = await client.responses.create({
  model: 'openai/gpt-5',
  agent_id: 'support-agent',
  input: 'Summarize this ticket',
  stream: true,
})

for await (const event of stream) {
  if (event.type === 'response.output_text.delta') {
    process.stdout.write(event.delta)
  }
}
```

**Returns:** `Promise<ResponsesStream>`.

For provider-native features, you can also pass `providerOptions` through unchanged:

```typescript
const response = await client.responses.create({
  model: 'openai/gpt-5',
  agent_id: 'support-agent',
  input: 'Continue this exchange',
  providerOptions: {
    openai: {
      previousResponseId: 'resp_123',
    },
  },
})
```

To request JSON object output, pass `text.format.type = 'json_object'`:

```typescript
const response = await client.responses.create({
  model: 'openai/gpt-5',
  agent_id: 'support-agent',
  input: 'Return a JSON object with issue, owner, and severity.',
  text: {
    format: {
      type: 'json_object',
    },
  },
})
```

For true structured output, pass `text.format.type = 'json_schema'` with the schema you want the agent to follow:

```typescript
const response = await client.responses.create({
  model: 'openai/gpt-5',
  agent_id: 'support-agent',
  input: 'Return a structured support ticket summary.',
  text: {
    format: {
      type: 'json_schema',
      name: 'ticket_summary',
      strict: true,
      schema: {
        type: 'object',
        properties: {
          summary: { type: 'string' },
          priority: { type: 'string' },
        },
        required: ['summary', 'priority'],
        additionalProperties: false,
      },
    },
  },
})
```

When the underlying provider returns provider-native response state, Mastra includes it back on the response as `providerOptions`. That lets you keep chaining provider-native state without leaving the Responses API surface:

```typescript
const first = await client.responses.create({
  model: 'openai/gpt-5',
  agent_id: 'support-agent',
  input: 'Tell me three facts about Saturn.',
})

const previousResponseId = first.providerOptions?.openai?.responseId

if (typeof previousResponseId !== 'string') {
  throw new Error('Expected the provider to return an OpenAI responseId')
}

const second = await client.responses.create({
  model: 'openai/gpt-5',
  agent_id: 'support-agent',
  input: 'Expand on the second fact.',
  providerOptions: {
    openai: {
      previousResponseId,
    },
  },
})
```

#### `stream(params)`

Creates a streaming response using the OpenAI-style helper.

```typescript
const stream = await client.responses.stream({
  model: 'openai/gpt-5',
  agent_id: 'support-agent',
  input: 'Summarize this ticket',
})
```

**Returns:** `Promise<ResponsesStream>`.

#### `retrieve(responseId, requestContext?)`

Retrieves a stored response by ID.

For stored responses, the ID is the persisted assistant turn ID returned as `response.id`.

```typescript
const response = await client.responses.retrieve('msg_123')

console.log(response.output_text)
```

**Returns:** `Promise<ResponsesResponse>`.

#### `delete(responseId, requestContext?)`

Deletes a stored response by ID.

```typescript
await client.responses.delete('msg_123')
```

**Returns:** `Promise<{ id: string; object: 'response'; deleted: true }>`.

## Response shape

The client SDK adds an `output_text` convenience field to non-streaming response objects and to the `response` payload on `response.created`, `response.in_progress`, and `response.completed` stream events. The underlying response object also includes OpenAI-style fields such as `completed_at`, `error`, `incomplete_details`, `tools`, token detail objects, and provider-returned `providerOptions` when the Mastra server provides them.

`tools` contains the configured tool definitions for the response. Executed tool activity is represented in `output`, where you can inspect `function_call`, `function_call_output`, and assistant `message` items in order.

For stored responses, `conversation_id` contains the raw Mastra thread ID for that response turn.

Streaming events preserve the server-provided `sequence_number` field and surface `response.content_part.*` plus `response.output_item.done` when the server emits them.

When tool activity is part of the response, `response.output_item.added` and `response.output_item.done` can carry non-message output items such as `function_call` alongside assistant `message` items.
