# Turso storage

`TursoStore` stores Mastra data in a local [Turso Database](https://github.com/tursodatabase/turso) file. It provides Mastra storage domains for memory, workflows, traces, scores, and other persisted data.

`TursoStore` doesn't provide vector storage. Configure a separate [vector store](/reference/vectors/vector-databases) when your application needs embeddings or vector search.

## Installation

Install the storage provider as a separate package:

```bash npm2yarn
npm install @mastra/turso@latest
```

## Usage

```typescript title="src/mastra/index.ts"
import { Mastra } from '@mastra/core/mastra'
import { TursoStore } from '@mastra/turso'

export const mastra = new Mastra({
  storage: new TursoStore({
    id: 'turso-storage',
    path: './mastra.db',
  }),
})
```

When you use the store without `Mastra`, call `init()` before accessing its storage domains:

```typescript
const storage = new TursoStore({
  id: 'turso-storage',
  path: './mastra.db',
})

await storage.init()

const memoryStore = await storage.getStore('memory')
```

`path` and `client` are mutually exclusive. The `client` option accepts a compatible `SqliteClient` and is intended for adapter integration and testing.

## Platform support

The package uses native bindings. Call `getTursoDatabaseSupport()` before constructing a store when your application needs to select a fallback.

| Platform | Architecture | Support |
| - | - | - |
| Linux with glibc | arm64 | Supported |
| Linux with glibc | x64 | Supported |
| macOS | arm64 | Supported |
| Windows | x64 | Supported |

Other combinations, including musl-based Linux and Intel macOS, aren't supported by `@tursodatabase/database@0.7.0`. Constructing `TursoStore` on an unsupported system returns an error before loading the native binding.

```typescript
import { getTursoDatabaseSupport, TursoStore } from '@mastra/turso'

const support = getTursoDatabaseSupport()

if (support.supported) {
  const storage = new TursoStore({
    id: 'turso-storage',
    path: './mastra.db',
  })
}
```

## Experimental features

Experimental Turso Database features are disabled by default. Enable a feature explicitly only when your application requires it:

```typescript
const storage = new TursoStore({
  id: 'multiprocess-storage',
  path: './mastra.db',
  experimental: ['multiprocess_wal'],
})
```

Experimental features can change between Turso Database releases. Test the selected features against your workload before using them in production.

## Local-file scope

`TursoStore` opens local Turso Database files. It doesn't connect to remote libSQL or hosted Turso databases. Use [libSQL storage](/reference/storage/libsql) for remote libSQL connections.
