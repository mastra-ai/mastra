## Schema Management

Schema creation and table management is handled automatically through each domain's `init()` method. When you call `storage.init()` (or when storage is initialized automatically), each domain creates its own tables:

**Memory Domain** (`memory`):
- `mastra_threads`: Stores conversation threads
- `mastra_messages`: Stores individual messages
- `mastra_resources`: Stores resource working memory data

**Workflows Domain** (`workflows`):
- `mastra_workflow_snapshot`: Stores workflow state and execution data

**Evals Domain** (`evals`):
- `mastra_scorers`: Stores scoring and evaluation data

**Observability Domain** (`observability`):
- `mastra_ai_spans`: Stores telemetry and tracing data

Each domain's `init()` method is called automatically when the storage instance initializes. In composite storage setups, only domains that are explicitly provided or exist in the default store will be initialized. This allows you to selectively enable domains based on your needs.

```typescript copy
// Initialize storage - all configured domains will create their tables
await storage.init();

// Or access a domain store directly - it will initialize if needed
const memoryStore = await storage.getStore('memory');
// Memory domain tables are now created
```
