# SignalProvider

Abstract base class for building signal providers. A signal provider monitors external sources (APIs, webhooks, event streams) and pushes notification signals into agent threads via the built-in subscription registry.

Signal providers are **not** processors by default. Providers that need to intercept agent execution can opt in by returning processors from `getInputProcessors()` / `getOutputProcessors()`. Providers that expose agent-callable tools can return them from `getTools()`.

## Usage example

Minimal polling provider that checks an API every 30 seconds:

```typescript title="src/signals/slack-signals.ts"
import { SignalProvider } from '@mastra/core/signals'
import type { SignalSubscription } from '@mastra/core/signals'

class SlackSignals extends SignalProvider<'slack-signals'> {
  readonly id = 'slack-signals'
  readonly pollInterval = 30_000

  async poll(subscriptions: SignalSubscription[]) {
    for (const sub of subscriptions) {
      const messages = await fetchSlackMessages(sub.externalResourceId)
      if (messages.length > 0) {
        await this.notify(
          {
            source: 'slack',
            kind: 'new-messages',
            summary: `${messages.length} new messages in ${sub.externalResourceId}`,
          },
          { threadId: sub.threadId, resourceId: sub.resourceId },
        )
      }
    }
  }
}
```

Register with an agent:

```typescript title="src/mastra/index.ts"
import { Agent } from '@mastra/core/agent'

const agent = new Agent({
  signals: [new SlackSignals()],
})
```

The agent automatically calls `connect(this)`, registers any processors/tools the provider returns, and starts polling.

## Constructor parameters

`SignalProvider` is abstract — subclasses call `super()` with no arguments.

## Methods

### Connection

#### `connect(agent)`

Called automatically by the Agent constructor. Establishes the bidirectional link so the provider can send signals back to the agent. Override to perform additional setup (always call `super.connect(agent)`).

```typescript prettier:false
class MySignals extends SignalProvider<"my-signals"> {
  readonly id = "my-signals";

  override connect(agent) {
    super.connect(agent);
    // Additional setup after agent link is established
  }
}
```

### Processor and tool integration

#### `getInputProcessors()`

Return input processors this provider needs registered with the agent. Override when your provider intercepts agent input steps (e.g., injecting context hints, detecting tool calls).

```typescript prettier:false
getInputProcessors() {
  return [this]; // when the provider itself implements processInputStep
}
```

Returns: `InputProcessorOrWorkflow[]`

#### `getOutputProcessors()`

Return output processors this provider needs registered with the agent. Override when your provider intercepts agent output steps.

```typescript prettier:false
getOutputProcessors() {
  return [this]; // when the provider itself implements processOutputStep
}
```

Returns: `OutputProcessorOrWorkflow[]`

#### `getTools()`

Return tools this provider exposes to the agent. Override when your provider adds agent-callable tools (e.g., subscribe/unsubscribe commands).

```typescript prettier:false
getTools() {
  return {
    subscribe_pr: createTool({ /* ... */ }),
    unsubscribe_pr: createTool({ /* ... */ }),
  };
}
```

Returns: `Record<string, unknown>`

### Subscription tracking

#### `subscribe(target, externalResourceId, metadata?)`

Subscribe a thread to an external resource. Protected method — call from within your provider implementation.

```typescript prettier:false
const sub = this.subscribe(
  { threadId: "thread-1", resourceId: "user-1" },
  "github:mastra-ai/mastra#123",
  { pr: 123 },
);
```

Returns: `SignalSubscription` — the created (or existing, with merged metadata) subscription.

#### `unsubscribe(target, externalResourceId)`

Remove a subscription. Returns `true` if removed, `false` if none existed.

```typescript prettier:false
const removed = this.unsubscribe(
  { threadId: "thread-1", resourceId: "user-1" },
  "github:mastra-ai/mastra#123",
);
```

Returns: `boolean`

#### `getSubscriptions()`

Get all active subscriptions for this provider.

```typescript prettier:false
const allSubs = this.getSubscriptions();
```

Returns: `SignalSubscription[]`

#### `getSubscriptionsForResource(externalResourceId)`

Get all subscriptions for a specific external resource.

```typescript prettier:false
const subs = this.getSubscriptionsForResource("github:mastra-ai/mastra#123");
for (const sub of subs) {
  await this.notify({ /* ... */ }, { threadId: sub.threadId, resourceId: sub.resourceId });
}
```

Returns: `SignalSubscription[]`

#### `getSubscriptionsForThread(target)`

Get all subscriptions for a specific thread.

```typescript prettier:false
const subs = this.getSubscriptionsForThread({
  threadId: "thread-1",
  resourceId: "user-1",
});
```

Returns: `SignalSubscription[]`

#### `hasSubscription(target, externalResourceId)`

Check if a subscription exists.

```typescript prettier:false
if (this.hasSubscription(target, "github:mastra-ai/mastra#123")) {
  // already subscribed
}
```

Returns: `boolean`

#### `unsubscribeAll(target)`

Remove all subscriptions for a thread.

```typescript prettier:false
const removed = this.unsubscribeAll({
  threadId: "thread-1",
  resourceId: "user-1",
});
```

Returns: `number` — count of removed subscriptions.

### Polling

#### `poll(subscriptions)`

Called on each poll cycle with all active subscriptions. Override to check external sources and emit notifications.

```typescript prettier:false
async poll(subscriptions: SignalSubscription[]) {
  for (const sub of subscriptions) {
    const events = await checkExternalSource(sub.externalResourceId);
    for (const event of events) {
      await this.notify(
        { source: "my-provider", kind: event.type, summary: event.message },
        { threadId: sub.threadId, resourceId: sub.resourceId },
      );
    }
  }
}
```

#### `startPolling()`

Start the polling timer. Called automatically by the Agent after `connect()`. Idempotent — calling multiple times has no effect.

```typescript prettier:false
provider.startPolling();
```

#### `stopPolling()`

Stop the polling timer.

```typescript prettier:false
provider.stopPolling();
```

### Webhooks

#### `handleWebhook(request)`

Handle an incoming webhook request. Override to parse the payload, match it to subscriptions, and emit notification signals.

```typescript prettier:false
async handleWebhook(request) {
  const payload = request.body as { repo: string; event: string };
  const subs = this.getSubscriptionsForResource(payload.repo);

  for (const sub of subs) {
    await this.notify(
      { source: "github", kind: payload.event, summary: `Event on ${payload.repo}` },
      { threadId: sub.threadId, resourceId: sub.resourceId },
    );
  }

  return { status: 200, body: { matched: subs.length } };
}
```

Returns: `Promise<{ status?: number; body?: unknown }>`

### Lifecycle

#### `start()`

Called after `connect()` to perform async initialization. Override for setup that requires the agent or Mastra to be available.

#### `stop()`

Called on shutdown. Default implementation stops polling and clears all subscriptions.

```typescript prettier:false
provider.stop();
```

### Notifications

#### `notify(notification, target)`

Send a notification signal to the connected agent. Protected convenience wrapper around `agent.sendNotificationSignal()`.

```typescript prettier:false
await this.notify(
  {
    source: "my-provider",
    kind: "pr-updated",
    summary: "PR #123 was updated",
    priority: "high",
    payload: { prNumber: 123 },
  },
  { threadId: "thread-1", resourceId: "user-1" },
);
```

## Type guard

### `isSignalProvider(obj)`

Runtime check for `SignalProvider` instances.

```typescript prettier:false
import { isSignalProvider } from "@mastra/core/signals";

if (isSignalProvider(obj)) {
  obj.connect(agent);
}
```

Returns: `boolean`
