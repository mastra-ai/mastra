# IAgentBuilder

`IAgentBuilder` is the runtime interface exposed by the Agent Builder. It is implemented by the EE-licensed `EditorAgentBuilder` class in `@mastra/editor/ee` and returned by `mastra.editor.resolveBuilder()`.

Server route handlers use it to read the resolved features, configuration, registry list, and any construction-time warnings produced by the Builder.

## Usage example

```typescript title="src/mastra/index.ts"
import { Mastra } from '@mastra/core'
import { MastraEditor } from '@mastra/editor'

const mastra = new Mastra({
  editor: new MastraEditor({
    builder: {
      enabled: true,
      configuration: {
        agent: {
          memory: { observationalMemory: true },
        },
      },
    },
  }),
})

const builder = await mastra.editor.resolveBuilder?.()
if (builder?.enabled) {
  const features = builder.getFeatures()
  const configuration = builder.getConfiguration()
  const warnings = builder.getModelPolicyWarnings?.() ?? []
}
```

## Methods

### `getFeatures()`

```typescript
interface IAgentBuilder {
  getFeatures(): AgentBuilderOptions['features']
}
```

Returns the resolved feature flags after construction-time validation. The shape mirrors `AgentBuilderOptions.features`, with any flags downgraded by the Builder (for example, `agent.browser` set to `false` when no provider is registered).

**Returns:** [`AgentBuilderOptions['features']`](/reference/editor/agent-builder/agent-builder-options).

### `getConfiguration()`

```typescript
interface IAgentBuilder {
  getConfiguration(): AgentBuilderOptions['configuration']
}
```

Returns the admin-pinned configuration object passed to `MastraEditor`. Returns the raw input — not server-resolved values like the resolved [`ModelPolicy`](/reference/editor/agent-builder/builder-model-policy).

**Returns:** [`AgentBuilderOptions['configuration']`](/reference/editor/agent-builder/agent-builder-options).

### `getRegistries()`

```typescript
interface IAgentBuilder {
  getRegistries?(): AgentBuilderOptions['registries']
}
```

Returns the opt-in skill registries the Builder is allowed to browse and install from. Returns `undefined` when no registries are configured.

**Returns:** [`AgentBuilderOptions['registries']`](/reference/editor/agent-builder/agent-builder-options) | `undefined`.

### `getModelPolicyWarnings()`

```typescript
interface IAgentBuilder {
  getModelPolicyWarnings?(): string[]
}
```

Returns warnings produced during construction-time validation (for example, allowlist entries with unknown providers that lack `kind: 'custom'`). Surfaced via `GET /editor/builder/settings.modelPolicyWarnings` for admin UI display.

**Returns:** `string[]` of human-readable warning messages.

## Resolution

`IAgentBuilder` is resolved by calling `mastra.editor.resolveBuilder()`, which:

1. Returns `undefined` if no `builder` config is present or `enabled` is `false`.
1. Validates the EE license. In production without a valid `MASTRA_EE_LICENSE`, this hard-throws.
1. Dynamic-imports `@mastra/editor/ee` and instantiates `EditorAgentBuilder`.

Subsequent calls return the cached instance.
