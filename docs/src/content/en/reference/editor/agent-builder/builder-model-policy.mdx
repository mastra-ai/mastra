# BuilderModelPolicy

`ModelPolicy` is the server-resolved, read-only shape served to the Agent Builder admin UI. The Builder produces it by combining the admin input on [`builder.configuration.agent.models`](/reference/editor/agent-builder/builder-models) with `features.agent.model`, then exposes it through `GET /editor/builder/settings.modelPolicy`.

`BuilderModelPolicy` is a deprecated alias for `ModelPolicy`. New code should import `ModelPolicy` directly.

See [Model policy](/docs/agent-builder/model-policy) for concepts and worked examples.

## Usage example

```typescript title="src/server/admin.ts"
import type { ModelPolicy } from '@mastra/core/agent-builder/ee'

async function fetchModelPolicy(): Promise<ModelPolicy> {
  const res = await fetch('/editor/builder/settings.modelPolicy')
  return res.json()
}
```

The admin UI uses the resolved `ModelPolicy` to render the allowlist editor, show whether the policy is currently active, and decide whether to expose the end-user model picker.

## Read-only

`ModelPolicy` is not directly writable. To change the resolved policy, update the admin input on `builder.configuration.agent.models` and restart the server. Construction-time validation warnings are surfaced via [`IAgentBuilder.getModelPolicyWarnings()`](/reference/editor/agent-builder/iagent-builder) and `GET /editor/builder/settings.modelPolicyWarnings`.
