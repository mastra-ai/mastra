# ToolProvider

The `ToolProvider` interface defines how the editor discovers and resolves integration tools from external platforms. Mastra ships with two built-in implementations: `ComposioToolProvider` and `ArcadeToolProvider`.

See [Tools](/docs/editor/tools) for a guide on setting up tool providers.

A `ToolProvider` may opt in to the user-driven surface used by the
Agent Builder and the agent editor. The legacy static surface above
keeps working; the methods below are all optional and gated by
`capabilities` flags.

### Storage

End-user connections persist in the `mastra_tool_provider_connections`
storage domain (composite primary key
`(authorId, providerId, connectionId)`). LibSQL, Clickhouse, and
Cloudflare KV all register the new table — no migration is required;
the table is created at boot.

### HTTP routes

All routes are mounted under `/api/tool-providers/` and require
authentication. `shared` connections are visible to every caller;
`per-author` and `caller-supplied` connections require ownership or
the `tool-providers:admin` permission.

- `POST   /:providerId/authorize`
- `GET    /:providerId/auth-status/:authId`
- `POST   /:providerId/connection-status`
- `GET    /:providerId/connections`
- `GET    /:providerId/connection-fields`
- `DELETE /:providerId/connections/:connectionId`
- `GET    /:providerId/connections/:connectionId/usage`
- `GET    /:providerId/health`

### Client SDK

`client.toolProviders` exposes the v1 surface for browser apps:

```typescript
import { MastraClient } from '@mastra/client-js'

const client = new MastraClient({ baseUrl: '/api' })
const provider = client.getToolProvider('composio')

await provider.authorize({ toolkit: 'gmail', scope: 'per-author' })
await provider.listConnections({ toolkit: 'gmail' })
await provider.disconnectConnection('ca_xxx')
```

## ComposioToolProvider

Connects to [Composio](https://composio.dev) for access to hundreds of integration tools.

### Usage example

```typescript title="src/mastra/index.ts"
import { MastraEditor } from '@mastra/editor'
import { ComposioToolProvider } from '@mastra/editor/providers/composio'

const editor = new MastraEditor({
  toolProviders: {
    composio: new ComposioToolProvider({
      apiKey: process.env.COMPOSIO_API_KEY!,
    }),
  },
})
```

### Constructor parameters

<PropertiesTable
  content={[
  {
    name: 'apiKey',
    type: 'string',
    description: 'Your Composio API key.',
  },
]}
/>

### Tool slugs

Composio tools use uppercase slug format: `GITHUB_CREATE_ISSUE`, `SLACK_SEND_MESSAGE`.

### Authentication

`ComposioToolProvider` implements the full v1 user-driven surface
(`authorize`, `getAuthStatus`, `listConnections`,
`listConnectionFields`, `getConnectionStatus`, `revokeConnection`,
`getHealth`) and reports the following capabilities:

- `multipleConnectionsPerToolkit: true`
- `batchConnectionStatus: true`
- `reauthorizeReusesConnectionId: true`
- `supportsRevoke: true`

Connections are bucketed by the connection's `scope`. For
`caller-supplied` scope, the runtime resolves the Composio
`internalUserId` from `ctx[MASTRA_RESOURCE_ID_KEY]`.

## ArcadeToolProvider

Connects to [Arcade](https://arcade.dev) for a curated tool catalog with built-in authentication.

### Usage example

```typescript title="src/mastra/index.ts"
import { MastraEditor } from '@mastra/editor'
import { ArcadeToolProvider } from '@mastra/editor/providers/arcade'

const editor = new MastraEditor({
  toolProviders: {
    arcade: new ArcadeToolProvider({
      apiKey: process.env.ARCADE_API_KEY!,
    }),
  },
})
```

### Constructor parameters

<PropertiesTable
  content={[
  {
    name: 'apiKey',
    type: 'string',
    description: 'Your Arcade API key.',
  },
  {
    name: 'baseURL',
    type: 'string',
    description: 'Custom base URL for the Arcade API.',
    isOptional: true,
  },
]}
/>

### Tool slugs

Arcade tools use `Toolkit.ToolName` format: `Github.GetRepository`, `Slack.SendMessage`.

### Authentication

Like Composio, tool execution uses `resourceId` from request context for per-user authorization. The provider maps that value to the user identity required by Arcade's auth flow.

`ArcadeToolProvider` currently exposes only the static surface
(`listToolkits`, `listTools`, `getToolSchema`, `resolveTools`). It does
not implement the v1 user-driven methods, so the connection picker UI
falls back to the static read-only flow for Arcade tools.
