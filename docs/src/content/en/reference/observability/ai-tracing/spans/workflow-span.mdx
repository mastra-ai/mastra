# Workflow Spans

Workflow spans track the execution of Mastra workflows, including individual steps, control flow operations, and state transitions.

## Span Types

### WORKFLOW_RUN

Represents the complete execution of a workflow from start to finish.

#### Attributes

<PropertiesTable
  props={[
    {
      name: "workflowId",
      type: "string",
      description: "Unique identifier for the workflow",
      required: true,
    },
    {
      name: "workflowName",
      type: "string",
      description: "Human-readable workflow name",
      required: true,
    },
    {
      name: "version",
      type: "string",
      description: "Workflow version",
      required: false,
    },
    {
      name: "trigger",
      type: "string",
      description: "How the workflow was triggered (e.g., 'manual', 'schedule', 'event')",
      required: false,
    },
    {
      name: "status",
      type: "string",
      description: "Final workflow status (e.g., 'completed', 'failed', 'suspended')",
      required: false,
    },
  ]}
/>

#### Example

```typescript
{
  type: 'WORKFLOW_RUN',
  name: 'customer-onboarding',
  startTime: new Date(),
  attributes: {
    workflowId: 'wf_onboarding_v2',
    workflowName: 'Customer Onboarding',
    version: '2.1.0',
    trigger: 'manual'
  },
  input: {
    customerId: 'cust_123',
    email: 'user@example.com',
    plan: 'premium'
  },
  output: {
    success: true,
    accountId: 'acc_456',
    completedSteps: 5
  },
  metadata: {
    status: 'completed',
    duration: 3500
  }
}
```

    },
    {

      required: false,

#### Example

```typescript

  name: 'parallel-validation',
  startTime: new Date(),

  }
}
```

## Common Patterns

### Nested Workflow Structure

Workflows create hierarchical span structures:

```
WORKFLOW_RUN
├── WORKFLOW_STEP (initialize)
├── WORKFLOW_CONDITIONAL (check-conditions)
│   └── WORKFLOW_CONDITIONAL_EVAL
├── WORKFLOW_PARALLEL (parallel-processing)
│   ├── WORKFLOW_STEP (branch-1)
│   └── WORKFLOW_STEP (branch-2)
├── WORKFLOW_LOOP (process-items)
│   ├── WORKFLOW_STEP (iteration-1)
│   └── WORKFLOW_STEP (iteration-2)
└── WORKFLOW_STEP (finalize)
```

### Error Handling

Track error recovery and compensation:

```typescript
{
  type: 'WORKFLOW_STEP',
  name: 'error-handler',
  attributes: {
    stepId: 'step_error_handler',
    stepName: 'Handle Error',
    stepType: 'compensation'
  },
  input: {
    error: {
      name: 'ValidationError',
      message: 'Invalid input data'
    }
  },
  output: {
    handled: true,
    action: 'rollback'
  }
}
```

### State Transitions

Track workflow state changes:

```typescript
metadata: {
  previousState: 'processing',
  currentState: 'suspended',
  stateTransition: 'suspend',
  suspensionReason: 'waiting_for_approval'
}
```

## Metadata Best Practices

### Performance Tracking

Include execution metrics:

```typescript
metadata: {
  executionTime: 1500, // ms
  queueTime: 200, // ms
  cpuTime: 1200, // ms
  memoryUsed: 45678 // bytes
}
```

### Business Context

Add business-relevant information:

```typescript
metadata: {
  customerId: 'cust_123',
  orderId: 'ord_456',
  priority: 'high',
  sla: '2h',
  cost: 0.05
}
```

### Debugging Information

Include data for troubleshooting:

```typescript
metadata: {
  retryAttempt: 2,
  maxRetries: 3,
  backoffDelay: 1000,
  errorCount: 1,
  warnings: ['Rate limit approaching']
}
```
