# SensitiveDataFilter

The `SensitiveDataFilter` processor automatically redacts sensitive information from span data before it's exported to observability platforms. It's enabled by default in Mastra's AI tracing configuration.

## Constructor

```typescript
constructor(sensitiveKeys?: string[])
```

Creates a new SensitiveDataFilter with specified sensitive field names.

## Default Sensitive Fields

When no custom fields are provided, the filter redacts these common sensitive fields:

- `password`
- `token`
- `secret`
- `key`
- `apiKey`
- `api_key`
- `auth`
- `authorization`
- `bearer`
- `jwt`
- `credential`
- `credentials`
- `sessionId`
- `session_id`
- `cookie`
- `cookies`

## How It Works

The SensitiveDataFilter:

1. **Scans recursively** through all span data
2. **Matches field names** case-insensitively
3. **Redacts values** with `[REDACTED]`
4. **Preserves structure** of the data
5. **Handles nested objects** and arrays

### Redaction Process

```typescript
// Before filtering
```

## Advanced Patterns

### Partial Redaction

For custom partial redaction, extend the SensitiveDataFilter:

```typescript
class PartialRedactionFilter extends SensitiveDataFilter {
  protected redactValue(key: string, value: any): any {
    if (key.toLowerCase().includes('email') && typeof value === 'string') {
      // Partially redact email: john@example.com -> j***@example.com
      const [user, domain] = value.split('@');
      if (domain) {
        return `${user[0]}***@${domain}`;
      }
    }

    if (key.toLowerCase().includes('phone') && typeof value === 'string') {
      // Show last 4 digits: 555-123-4567 -> ***-***-4567
      return value.replace(/\d(?=\d{4})/g, '*');
    }

    // Default redaction for other fields
    return '[REDACTED]';
  }
}
```

### Conditional Redaction

Redact based on environment or context:

```typescript
class ConditionalRedactionFilter extends SensitiveDataFilter {
  process(span: AnyAISpan): AnyAISpan | null {
    // Only redact in production
    if (process.env.NODE_ENV !== 'production') {
      return span;
    }

    // Only redact external traces
    if (span.attributes?.internal === true) {
      return span;
    }

    return super.process(span);
  }
}
```

### Allowlist Pattern

Only allow specific fields:

```typescript
class AllowlistFilter implements AISpanProcessor {
  name = 'allowlist-filter';

  constructor(private allowedFields: Set<string>) {}

  process(span: AnyAISpan): AnyAISpan | null {
    const filtered = this.filterObject(span);
    return filtered as AnyAISpan;
  }

  private filterObject(obj: any): any {
    if (!obj || typeof obj !== 'object') return obj;

    const filtered: any = {};
    for (const [key, value] of Object.entries(obj)) {
      if (this.allowedFields.has(key.toLowerCase())) {
        filtered[key] = Array.isArray(value)
          ? value.map(v => this.filterObject(v))
          : this.filterObject(value);
      }
    }
    return filtered;
  }

  async shutdown(): Promise<void> {}
}
```

## What Gets Filtered

The SensitiveDataFilter processes:

- **Span attributes** - Core span properties
- **Span metadata** - Additional context data
- **Input data** - Request/input parameters
- **Output data** - Response/output values
- **Error details** - Error messages and stack traces
- **Custom fields** - Any additional span data

## Performance Considerations

<Callout type="info">
  The SensitiveDataFilter operates synchronously on each span with minimal overhead. For high-volume applications, consider:

  1. Limiting the number of sensitive fields to check
  2. Using simpler field names (avoid complex patterns)
  3. Filtering at the source when possible
</Callout>

## Combining with Other Processors

Processors are executed in order:

```typescript
export const mastra = new Mastra({
  observability: {
    configs: {
      production: {
        serviceName: 'my-service',
        processors: [
          // First: Redact sensitive data
          new SensitiveDataFilter(['password', 'apiKey']),

          // Second: Enrich with metadata
          new MetadataEnrichmentProcessor(),

          // Third: Filter by criteria
          new SpanFilterProcessor(span => span.duration > 100)
        ],
        exporters: [...]
      }
    }
  }
});
```

```typescript
  input: { apiKey: 'sk-test123' }
```

### 3. Monitor for Leaks

Periodically review exported traces for sensitive data:

```typescript
class LeakDetector implements AISpanProcessor {
  private patterns = [
    /sk-[a-zA-Z0-9]{48}/,  // OpenAI API keys
    /\b\d{3}-\d{2}-\d{4}\b/, // SSN pattern
    /\b\d{16}\b/,            // Credit card pattern
  ];

  process(span: AnyAISpan): AnyAISpan | null {
    const stringified = JSON.stringify(span);

    for (const pattern of this.patterns) {
      if (pattern.test(stringified)) {
        console.error('Potential sensitive data leak detected!');
        // Log incident for review
      }
    }

    return span;
  }
}
```
