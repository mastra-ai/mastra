# 贡献模板

Mastra 社区在创建展示创新应用模式的模板方面发挥着至关重要的作用。本指南解释了如何为您自己的模板贡献到 Mastra 生态系统。

## 模板贡献流程

### 1. 查看要求

在创建模板之前，请确保您了解：

- [模板参考](/reference/templates/overview) - 技术要求和约定
- [项目结构](/docs/cn/getting-started/project-structure) - 标准 Mastra 项目组织
- 社区指南和质量标准

### 2. 开发您的模板

按照已建立的模式创建您的模板：

- 专注于特定的用例或模式
- 包含全面的文档
- 使用全新安装进行彻底测试
- 遵循所有技术要求
- 确保 github 仓库是模板仓库。[如何创建模板仓库](https://docs.github.com/en/repositories/creating-and-managing-repositories/creating-a-template-repository)

### 3. 提交审核

一旦您的模板准备好，通过我们的贡献表单提交。模板需要经过审核流程以确保质量和一致性。

## 提交指南

### 模板标准

我们接受符合以下条件的模板：

- **展示独特价值** - 展示现有模板未涵盖的创新用例或模式
- **遵循约定** - 遵守所有技术要求和结构指南
- **包含高质量文档** - 提供清晰的设置说明和使用示例
- **可靠工作** - 安装后以最少的设置即可正确运行

### 质量标准

模板必须满足以下质量基准：

- **代码质量** - 干净、有注释且可维护的代码
- **错误处理** - 对外部 API 和用户输入进行适当的错误处理
- **类型安全** - 完整的 TypeScript 类型定义，在适当的地方使用 Zod 验证
- **文档** - 包含设置和使用说明的综合性 README
- **测试** - 验证可以与全新安装一起工作

## 提交流程

### 1. 准备您的模板

确保您的模板符合[模板参考](/reference/templates/overview)中概述的所有要求：

- `src/mastra/` 目录中的正确项目结构
- 标准 TypeScript 配置
- 全面的 `.env.example` 文件
- 包含设置说明的详细 README

### 2. 提交您的模板

使用我们的贡献表单提交您的模板：

**[提交模板贡献](https://forms.gle/g1CGuwFxqbrb3Rz57)**

### 必需信息

提交模板时，请提供：

- **模板名称** - 清晰描述用例的名称
- **模板作者名称** - 您的姓名或组织名称
- **模板作者电子邮件** - 有关您提交的通信联系电子邮件
- **GitHub URL** - 指向您模板仓库的链接
- **描述** - 模板功能及其价值的详细说明
- **可选图片** - 显示模板运行状态的屏幕截图或图表
- **可选演示视频** - 展示模板功能的视频链接

## 审核流程

### 审核标准

模板根据以下标准进行评估：

- **技术合规性** - 遵守模板规则和约定
- **代码质量** - 干净、可维护且有良好文档的代码
- **独特性** - 新颖的用例或创新的实现模式
- **教育价值** - 有效教授 Mastra 概念的能力
- **社区利益** - 对更广泛的 Mastra 社区的潜在价值

### 反馈和迭代

如果您的模板需要改进：

- 您将收到有关所需更改的具体反馈
- 进行请求的修改并重新提交
- 审核流程持续进行，直到模板符合标准

## 社区指南

### 模板创意

考虑创建以下模板：

- **特定行业用例** - 医疗保健、金融、教育等
- **集成模式** - 特定 API 或服务集成
- **高级技术** - 复杂工作流、多智能体系统或新颖模式
- **学习资源** - 特定概念的循序渐进教程

### 开发最佳实践

- **从简单开始** - 从最小的工作示例开始，逐步增加复杂性
- **彻底记录** - 包含详细的注释和全面的 README
- **广泛测试** - 验证您的模板在不同环境中工作
- **寻求反馈** - 在提交之前与社区分享以获取早期反馈

### 社区参与

- **加入 Discord** - 参与 [Mastra Discord 社区](https://discord.gg/BTYqqHKUrf)
- **分享进度** - 向社区更新您的模板开发情况
- **帮助他人** - 协助其他贡献者处理他们的模板
- **保持更新** - 跟踪新的 Mastra 功能和约定

## 模板维护

### 持续责任

作为模板贡献者，您可能被要求：

- **更新依赖项** - 使模板与最新的 Mastra 版本保持同步
- **修复问题** - 解决错误或兼容性问题
- **改进文档** - 根据用户反馈增强说明
- **添加功能** - 用新功能扩展模板

### 社区支持

Mastra 团队和社区提供：

- **技术指导** - 帮助解决复杂的实施挑战
- **审核反馈** - 详细反馈以提高模板质量
- **推广** - 向社区展示已批准的模板
- **维护协助** - 支持保持模板更新

## 验证清单

提交模板前，请验证：

- [ ] 所有代码组织在 `src/mastra/` 目录中
- [ ] 使用标准的 Mastra TypeScript 配置
- [ ] 包含全面的 `.env.example`
- [ ] 包含设置说明的详细 README
- [ ] 没有 monorepo 或 web 框架样板文件
- [ ] 全新安装和环境设置后成功运行
- [ ] 遵循所有代码质量标准
- [ ] 展示清晰、有价值的用例

## 社区展示

### 模板画廊

已批准的模板将在以下位置展示：

- **mastra.ai/templates** - 社区模板画廊（即将推出）
- **文档** - 在相关文档部分中引用
- **社区精选** - 在新闻通讯和社区更新中精选

### 认可

模板贡献者将获得：

- **署名** - 您的姓名和联系信息与模板一起展示
- **社区认可** - 在社区渠道中的致谢

## 开始使用

准备好贡献模板了吗？

1. **探索现有模板** - 回顾当前模板以获取灵感和模式
2. **规划您的模板** - 定义用例和价值主张
3. **遵循要求** - 确保符合所有技术要求
4. **构建和测试** - 创建一个工作良好且有文档的模板
5. **提交审核** - 使用贡献表单提交您的模板

您的贡献有助于发展 Mastra 生态系统，并为整个社区提供宝贵的资源。我们期待看到您创新的模板！
