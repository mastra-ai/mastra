# 许可证

## Apache 许可证 2.0

Mastra 采用 Apache 许可证 2.0 许可，这是一种宽松的开源许可证，为用户提供了广泛的使用、修改和分发软件的权利。

### 什么是 Apache 许可证 2.0？

Apache 许可证 2.0 是一种宽松的开源许可证，授予用户广泛的权利来使用、修改和分发软件。它允许：

- 任何目的的免费使用，包括商业使用
- 查看、修改和重新分发源代码
- 创建和分发衍生作品
- 无限制的商业使用
- 贡献者的专利保护

Apache 许可证 2.0 是最宽松和商业友好的开源许可证之一。

### 为什么我们选择 Apache 许可证 2.0

我们选择 Apache 许可证 2.0 有以下几个重要原因：

1. **真正的开源**：它是一种公认的开源许可证，符合开源原则和社区期望。

2. **商业友好**：它允许无限制的商业使用和分发，适合各种规模的企业。

3. **专利保护**：它包含对用户的明确专利保护，提供额外的法律保障。

4. **社区重点**：它鼓励社区贡献和协作，无限制。

5. **广泛采用**：它是业内最受欢迎和最被理解的开源许可证之一。

### 使用 Mastra 构建您的业务

Apache 许可证 2.0 为使用 Mastra 构建业务提供了最大的灵活性：

#### 允许的商业模式

- **构建应用程序**：创建和销售使用 Mastra 构建的应用程序
- **提供咨询服务**：提供专业知识、实施和定制服务
- **开发定制解决方案**：使用 Mastra 为客户构建定制的 AI 解决方案
- **创建插件和扩展**：开发和销售扩展 Mastra 功能的互补工具
- **培训和教育**：提供关于有效使用 Mastra 的课程和教育材料
- **托管服务**：提供 Mastra 作为托管或管理服务
- **SaaS 平台**：构建由 Mastra 驱动的 SaaS 平台

#### 合规使用示例

- 一家公司使用 Mastra 构建 AI 驱动的客户服务应用程序并将其销售给客户
- 一家咨询公司提供 Mastra 的实施和定制服务
- 一位开发人员使用 Mastra 创建专业智能体和工具，并将其许可给其他企业
- 一家初创公司构建由 Mastra 驱动的垂直特定解决方案（例如，医疗保健 AI 助手）
- 一家公司向其客户提供 Mastra 作为托管服务
- 一家 SaaS 平台将 Mastra 集成为其 AI 后端

#### 合规要求

Apache 许可证 2.0 要求很少：

- **署名**：保留版权声明和许可证信息（包括 NOTICE 文件）
- **状态更改**：如果您修改了软件，请说明您已进行更改
- **包含许可证**：分发时包含 Apache 许可证 2.0 的副本
