# 认证概述

Mastra 允许您选择处理身份验证的方式，以便使用适合您技术栈的身份系统来保护应用程序端点的访问。

您可以从简单的共享密钥 JWT 身份验证开始，当需要更高级的身份功能时，切换到 Supabase、Firebase Auth、Auth0、Clerk 或 WorkOS 等提供程序。

## 默认行为

身份验证在 Mastra 中是可选的。当您配置身份验证时：

- **所有内置 API 路由**（`/api/agents/*`、`/api/workflows/*` 等）默认需要身份验证
- **自定义 API 路由**默认也需要身份验证
- **公共访问**可以在自定义路由上使用 `requiresAuth: false` 启用

如果未配置身份验证，所有路由都可以公开访问。

有关控制自定义端点上的身份验证，请参阅[自定义 API 路由](/docs/cn/server/custom-api-routes#authentication)。
