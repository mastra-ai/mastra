# 使用 Web 框架部署

当 Mastra 与 Web 框架集成时，它使用框架的标准部署流程与您的应用程序一起部署。请按照以下说明确保您的 Mastra 集成正确部署。

:::warning

如果您要部署到云提供商，请从您的 Mastra 配置中删除任何 [LibSQLStore](/reference/storage/libsql) 的使用。LibSQLStore 需要文件系统访问权限，与无服务器平台不兼容。

:::

集成指南：

- [与 Next.js 一起使用](/guides/getting-started/next-js)
- [与 Astro 一起使用](/guides/getting-started/astro)

## 在 Vercel 上使用 Next.js

如果您已[按照我们的指南](//guides/getting-started/next-js)将 Mastra 与 Next.js 集成，并计划部署到 Vercel，请将 `serverExternalPackages: ["@mastra/*"]` 添加到您的 `next.config.ts`：

```typescript {4} title="next.config.ts"
import type { NextConfig } from "next";

const nextConfig: NextConfig = {
  serverExternalPackages: ["@mastra/*"],
};

export default nextConfig;
```

## 在 Vercel 上使用 Astro

如果您已[按照我们的指南](/guides/getting-started/astro)将 Mastra 与 Astro 集成，并计划部署到 Vercel，请将 Vercel 适配器和服务器输出添加到您的 `astro.config.mjs`：

```javascript {2,5-6} title="astro.config.mjs"
import { defineConfig } from "astro/config";
import vercel from "@astrojs/vercel";

export default defineConfig({
  adapter: vercel(),
  output: "server",
});
```

## 在 Netlify 上使用 Astro

如果您已[按照我们的指南](/guides/getting-started/astro)将 Mastra 与 Astro 集成，并计划部署到 Netlify，请将 Netlify 适配器和服务器输出添加到您的 `astro.config.mjs`：

```javascript {2,5-6} title="astro.config.mjs"
import { defineConfig } from "astro/config";
import netlify from "@astrojs/netlify";

export default defineConfig({
  adapter: netlify(),
  output: "server",
});
```
