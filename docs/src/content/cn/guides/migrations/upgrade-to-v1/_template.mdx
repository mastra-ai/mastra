这是一个关于如何填充各个迁移指南的模板样式指南。使用此作为如何组织本目录中文件的指导。您不能偏离此结构。

请遵循 `.cursor/rules/writing-documentation.mdc` 中的说明来撰写正文内容。

在找到所有需要记录的更改后，您应该将它们分为两大组："变更"和"移除"。

示例：

```md
## 变更

## 移除
```

所有更改都必须放在这两个部分下。

每个更改（无论是"变更"还是"移除"）都应该是 h3 标题。您必须完全遵循这种格式。

示例：

```md
## 变更

### `oldFunction` 改为 `newFunction`
```

现在标题已经就位，您需要为每个更改遵循完全精确的结构。您不能偏离此结构。

用一到两句话简要描述更改。接下来，用一句话解释为什么更改了它。

每个更改的第三个元素必须是关于如何将现有代码迁移到新版本的解释。保持简洁清晰，尽量不要省略太多信息。之后必须提供一个使用"diff"语法的代码块。在代码块内部，您需要展示需要更改的代码的"之前"和"之后"示例，并使用 diff 语法。

示例：

````md
## 变更

### `oldFunction` 改为 `newFunction`

`oldFunction` 已重命名为 `newFunction`。此更改提高了 API 的清晰度和一致性。

要迁移，请在代码库中将所有 `oldFunction` 的实例替换为 `newFunction`。

```diff
- const result = oldFunction(args);
+ const result = newFunction(args);
```
````

如果迁移需要多个步骤而不仅仅是简单的查找和替换，您可以在代码块之前添加额外的解释。如果需要多个步骤，请使用编号列表清楚地解释每个步骤。
