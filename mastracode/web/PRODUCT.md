# Product

## Register

product

## Users

Developers working in local repositories through MastraCode's web, desktop, and terminal surfaces. They need to move quickly between projects, threads, models, and provider accounts without losing context or leaving their coding workflow.

## Product Purpose

MastraCode provides one agent coding experience across web, Electron, and the terminal. The shared interface should make local projects and authenticated Claude or Codex models immediately usable, while the SDK remains the source of truth for runtime behavior.

## Brand Personality

Focused, precise, and quietly distinctive. The product should feel fast and technically capable, with moments of polish that reinforce progress without competing with the work.

## Anti-references

Avoid marketing-page composition inside the product, decorative dashboard cards, terminal cosplay, cinematic startup sequences, and motion that delays access to the workspace. Desktop must not feel like a remote site wrapped in a browser frame.

## Design Principles

- Keep the developer's work as the visual priority.
- Share product UI and behavior across hosts; specialize only at native boundaries.
- Make system state visible and actionable without adding noise.
- Use motion to communicate progress and continuity, never to make users wait.
- Preserve local-first behavior and a complete interface when the network is unavailable.

## Accessibility & Inclusion

Target WCAG 2.2 AA for text, controls, focus states, and status communication. All workflows remain keyboard accessible, status changes use semantic live regions, color is never the only signal, and every nonessential animation has a reduced-motion alternative.
