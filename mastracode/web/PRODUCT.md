# Product

## Register

product

## Users

Software developers and engineering teams working in a browser-based coding workspace. They use Mastra Code to understand code, plan and implement changes, diagnose failures, review work, and keep long-running agent sessions connected to a Factory, repository, branch, and workspace.

## Product Purpose

Mastra Code turns a development workspace into a persistent conversational work surface. It should help users move from intent to verified code changes while keeping project context available without making infrastructure metadata the focus. Success means a new conversation feels immediately approachable, an active conversation stays legible, and operational details remain easy to find when they matter.

## Brand Personality

Calm, capable, and direct. The interface should feel familiar to users of leading general-purpose AI assistants while retaining Mastra's identity and the confidence of a production tool.

## Anti-references

Terminal-themed welcome screens, metadata-first developer dashboards, dense diagnostic readouts in the primary conversation path, ornamental card grids, and empty states that feel like internal tooling. Avoid decorative effects or technical jargon that make the user think about the implementation before their task.

## Design Principles

- Lead with the user's intent; reveal workspace details progressively.
- Use familiar conversational affordances so the interface disappears into the task.
- Keep hierarchy quiet and deliberate; one clear next action beats competing controls.
- Preserve useful project context without presenting it as a status report.
- Prefer concise, human language over implementation terminology.

## Accessibility & Inclusion

Target WCAG 2.2 AA. Support complete keyboard navigation, visible focus states, readable contrast in light and dark themes, reduced-motion preferences, semantic structure, and labels that do not rely on color or iconography alone.
