## Use Tailwind v4 CSS-First APIs

Tailwind v4 is configured in CSS. In this repo, `packages/playground-ui/src/index.css` imports Tailwind and `packages/playground-ui/theme.css`, and `theme.css` ships the raw `@theme` contract for consumers. Prefer v4 directives and CSS variables instead of reintroducing v3-era JavaScript config or config-derived theme reads.

**Use these v4 APIs:**

- `@import 'tailwindcss'` instead of `@tailwind base/components/utilities`
- `@theme` for shared tokens that should generate utilities or variants
- `@utility` for custom utilities that should work with variants like `hover:`, `focus:`, and responsive prefixes
- `@custom-variant` for project-wide variants such as the app's class-based dark mode selector
- `@variant` when applying a Tailwind variant inside custom CSS
- `@source` and `@source inline(...)` for explicit source registration or safelisting
- `@reference` when a separately bundled stylesheet needs access to theme variables, custom utilities, or custom variants without duplicating CSS

**Incorrect:**

```css
/* DON'T: Add v3 Tailwind directives */
@tailwind base;
@tailwind components;
@tailwind utilities;

/* DON'T: Define variant-capable custom utilities in @layer */
@layer utilities {
  .surface-panel {
    background: var(--surface2);
  }
}
```

```ts
// DON'T: Reintroduce config-derived theme reads for runtime styling
import resolveConfig from 'tailwindcss/resolveConfig';
```

**Correct:**

```css
@import 'tailwindcss';

@utility surface-panel {
  background: var(--surface2);
  border-color: var(--border1);
}

@custom-variant dark (&:is(.dark *));
```

```tsx
// DO: Use CSS variables directly when JavaScript needs a theme value
<motion.div animate={{ backgroundColor: 'var(--color-surface4)' }} />
```

**v4 gotchas to remember:**

- JavaScript config files are legacy compatibility only and are not auto-detected; do not add `tailwind.config.ts` for new work.
- `theme()` is deprecated for ordinary CSS; use CSS theme variables instead.
- `@config` and `@plugin` are compatibility tools. Use them only when the package already depends on that integration.
- `@layer` is now the native CSS cascade layer rule; it is not the replacement for custom utilities.
- Default `border`, `divide`, `ring`, and outline colors are less opinionated in v4. Specify DS colors such as `border-border1` and `focus-visible:outline-accent1` when the color matters.
- Use the v4 important modifier suffix (`bg-surface4!`) instead of the deprecated prefix form.
- Stacked variants apply left to right in v4; keep order-sensitive child and typography variants explicit.
- The `hover` variant only applies on hover-capable primary input devices. Do not rely on hover for required touch behavior.
- Individual transform utilities use individual CSS properties. If you write a custom transition property list, include `scale`, `rotate`, or `translate` instead of assuming `transform` covers them.
